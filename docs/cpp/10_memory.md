# 메모리 구조와 관리

## 아두이노의 메모리 종류

| 종류 | ESP32-C3 | 아두이노 우노 | 용도 |
|------|----------|--------------|------|
| **플래시** | 4MB | 32KB | 코드 저장 (프로그램) |
| **SRAM** | 400KB | 2KB | 실행 중 변수, 스택 |
| **EEPROM/NVS** | 4MB (NVS) | 1KB | 전원 꺼져도 유지 |

---

## SRAM 구조

SRAM은 4개 영역으로 나뉩니다.

```
높은 주소
┌──────────┐
│  스택    │  ← 함수 호출, 지역변수 (위에서 아래로 자람)
│  ↓       │
│          │
│  ↑       │
│  힙      │  ← 동적 할당 new/malloc (아래에서 위로 자람)
├──────────┤
│  BSS     │  ← 초기화되지 않은 전역/static 변수
├──────────┤
│  데이터   │  ← 초기화된 전역/static 변수
└──────────┘
낮은 주소
```

스택과 힙이 서로 만나면 **스택 오버플로우** 발생 → 오작동, 리셋

## 스택 (Stack)

- 함수 호출 시 자동으로 생성, 함수 종료 시 자동 해제
- 지역변수, 함수 매개변수가 여기에 저장됨
- 크기가 제한되어 있어서 **깊은 재귀나 큰 지역 배열은 위험**

```cpp
void 위험한함수() {
    int 큰배열[500];    // 스택에 1000바이트 — 우노에서는 위험!
    // ESP32는 괜찮음
}
```

## 힙 (Heap)

- `new`(C++) 또는 `malloc`(C)으로 수동 할당
- 사용 후 반드시 `delete`/`free`로 해제해야 함
- 해제 안 하면 **메모리 누수** 발생

```cpp
// C++ 방식
int* p = new int(42);     // 힙에 int 하나 할당
int* arr = new int[10];   // 힙에 int 배열 할당

delete p;                 // 반드시 해제
delete[] arr;             // 배열은 delete[]

// C 방식
int* p2 = (int*)malloc(sizeof(int) * 10);
free(p2);
```

> 아두이노에서는 동적 할당을 **가능하면 피하세요.**
> 메모리 단편화로 인해 나중에 할당이 실패하거나 불안정해집니다.
> 전역 배열이나 static 변수를 쓰는 게 낫습니다.

## String이 메모리 문제를 일으키는 이유

`String` 클래스는 내부적으로 힙을 사용합니다.
짧은 코드에서는 괜찮지만, `loop()`에서 반복적으로 String을 만들고 버리면
**힙 단편화**가 누적되어 수 시간 후 오작동할 수 있습니다.

```cpp
// 우노에서 위험한 패턴 — loop에서 String 계속 생성
void loop() {
    String 메시지 = "온도: " + String(온도) + "°C";  // 힙 할당/해제 반복
    Serial.println(메시지);
    delay(1000);
}

// 안전한 패턴 — 고정 크기 버퍼 사용
void loop() {
    char 버퍼[30];
    snprintf(버퍼, sizeof(버퍼), "온도: %.1f°C", 온도);
    Serial.println(버퍼);
    delay(1000);
}
```

> ESP32는 RAM이 400KB라 String을 써도 크게 문제없지만, 습관적으로 char[]를 쓰면 더 안정적입니다.

```cpp
// 남은 힙 메모리 확인 (ESP32)
Serial.print("여유 힙: ");
```

## volatile — 인터럽트와 공유 변수

인터럽트 함수와 일반 코드가 같은 변수를 공유할 때, 컴파일러 최적화로 인해
변수가 캐시에서 읽혀 최신값이 반영되지 않을 수 있습니다.
`volatile`은 컴파일러에게 "이 변수는 항상 메모리에서 직접 읽어라"고 지시합니다.

```cpp
volatile bool 버튼눌림 = false;  // 인터럽트와 공유하는 변수

void IRAM_ATTR 인터럽트핸들러() {
    버튼눌림 = true;   // 인터럽트에서 수정
}

void loop() {
    if (버튼눌림) {     // 메인 코드에서 읽기
        버튼눌림 = false;
        // 처리...
    }
}
```
