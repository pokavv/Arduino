# BME280 — 기압 / 온도 / 습도 복합 센서

## 개요

BME280은 Bosch Sensortec에서 제조한 고정밀 환경 복합 센서로,
기압(Pressure), 온도(Temperature), 상대 습도(Relative Humidity)를 하나의 패키지에서 측정합니다.
I2C 또는 SPI 인터페이스를 지원하며, 내부 보정 계수로 온도·기압·습도를 자동 보정합니다.

**측정 원리**
- 온도: 내장 PTC 저항 변화로 측정 (기압·습도 보정에도 사용)
- 기압: MEMS 피에조저항 방식 (실리콘 다이어프램이 기압에 따라 변형)
- 습도: 폴리머 커패시터 방식 (수분 흡수에 따른 정전용량 변화)

**아두이노에서의 활용**
고도 측정(GPS 보조), 실내 공기 질 모니터링, 날씨 스테이션, IoT 환경 센서 노드,
드론·기구의 기압 고도계 등 매우 폭넓게 활용됩니다.

## 핀 구성

### 일반 모듈 (6핀 또는 4핀 브레이크아웃 보드)

| 핀 이름 | 역할 |
|---------|------|
| VCC / VIN | 전원 (+3.3V, 일부 모듈은 5V 입력 후 내부 레귤레이터) |
| GND | 접지 |
| SCL / SCK | I2C 클록 / SPI 클록 |
| SDA / SDI | I2C 데이터 / SPI MOSI |
| CSB / CS | SPI 칩 셀렉트 (I2C 모드에서는 VCC 연결) |
| SDO / ADDR | SPI MISO / I2C 주소 선택 (GND=0x76, VCC=0x77) |

### IC 패키지 (LGA-8)

```
  ┌───────────────┐
  │ 1:SDO  5:VDDIO│
  │ 2:CSB  6:SDI  │
  │ 3:VDD  7:SCK  │
  │ 4:GND  8:SDO  │  (EP: GND)
  └───────────────┘
```

## 통신 프로토콜

### I2C 모드

**I2C 주소**
```
SDO/ADDR 핀:
  LOW  (GND 연결) → 0x76
  HIGH (VCC 연결) → 0x77  ← 기본값 (대부분 모듈)
```

**레지스터 구조 (주요)**
```
0xD0: id         → 0x60 (BME280 칩 ID)
0xE0: reset      → 0xB6 쓰면 소프트 리셋
0xF2: ctrl_hum   → 습도 오버샘플링 설정
0xF3: status     → 측정 중 상태
0xF4: ctrl_meas  → 온도/기압 오버샘플링 + 모드 설정
0xF5: config     → 대기 시간, IIR 필터, SPI 설정
0xF7~0xFC: press_msb ~ temp_xlsb → 원시 데이터
0xFD~0xFE: hum_msb, hum_lsb      → 습도 원시 데이터
```

**동작 모드**
```
00: Sleep Mode    — 측정 중지, 저전력 대기
01/10: Forced Mode — 1회 측정 후 자동 Sleep 복귀
11: Normal Mode   — 설정 간격으로 연속 측정
```

### SPI 모드

```
CSB 핀을 LOW로 유지하면 SPI 모드 활성화.
4선 SPI: SCK, SDI(MOSI), SDO(MISO), CS
3선 SPI: 지원하지만 일반적으로 사용 드묾.
```

## 아두이노 연결 방법

### I2C 연결 (ESP32-C3)

```
ESP32-C3          BME280 모듈
  3.3V ──────────── VCC
  GND  ──────────── GND
  G6   ──────────── SCL (I2C 클록)
  G5   ──────────── SDA (I2C 데이터)
  (SDO/ADDR → GND: 주소 0x76)
  (SDO/ADDR → VCC: 주소 0x77)  ← 기본값
```

> ESP32-C3의 기본 I2C 핀: SDA=G8, SCL=G9 (Wire.begin() 기본)
> 다른 핀 사용 시: `Wire.begin(SDA핀, SCL핀)` 명시

### SPI 연결 (ESP32-C3)

```
ESP32-C3          BME280 모듈
  3.3V ──────────── VCC
  GND  ──────────── GND
  G10  ──────────── CS  (칩 셀렉트)
  G6   ──────────── SCK (SPI 클록)
  G7   ──────────── SDI (MOSI)
  G2   ──────────── SDO (MISO)
```

### 라이브러리 설치

Arduino IDE > 라이브러리 매니저에서:
- `Adafruit BME280 Library` by Adafruit 설치
- 의존성: `Adafruit Unified Sensor`, `Adafruit BusIO` 함께 설치

또는:
- `BME280` by Tyler Glenn 설치 (더 가벼운 대안)

### I2C 코드 예시 (Adafruit)

```cpp
#include <Wire.h>
#include <Adafruit_Sensor.h>
#include <Adafruit_BME280.h>

#define SEA_LEVEL_HPA 1013.25  // 해수면 기압 (현지 기준 보정 필요)

Adafruit_BME280 bme;

void setup() {
  Serial.begin(115200);
  Wire.begin(5, 6); // SDA=G5, SCL=G6 (ESP32-C3 커스텀 핀)

  if (!bme.begin(0x76)) { // I2C 주소 0x76 또는 0x77
    Serial.println("BME280을 찾을 수 없습니다! 배선 확인!");
    while (1) delay(10);
  }

  // 날씨 모니터링 설정 (저전력, 느린 측정)
  bme.setSampling(Adafruit_BME280::MODE_NORMAL,
                  Adafruit_BME280::SAMPLING_X1,  // 온도
                  Adafruit_BME280::SAMPLING_X1,  // 기압
                  Adafruit_BME280::SAMPLING_X1,  // 습도
                  Adafruit_BME280::FILTER_OFF,
                  Adafruit_BME280::STANDBY_MS_1000);
}

void loop() {
  float temperature = bme.readTemperature();          // °C
  float pressure    = bme.readPressure() / 100.0F;    // hPa
  float humidity    = bme.readHumidity();              // %
  float altitude    = bme.readAltitude(SEA_LEVEL_HPA); // m

  Serial.printf("온도: %.2f°C  기압: %.2f hPa  습도: %.2f%%  고도: %.1f m\n",
                temperature, pressure, humidity, altitude);

  delay(2000);
}
```

## 측정값 계산 공식

### 고도 계산 (국제 표준 대기 공식)

```
고도(m) = 44330 × (1 - (P / P0)^(1/5.255))

  P  = 현재 기압 (hPa)
  P0 = 해수면 기준 기압 (hPa, 기본 1013.25)

예: P = 990 hPa, P0 = 1013.25 hPa
  고도 = 44330 × (1 - (990/1013.25)^0.1903) ≈ 194 m
```

### 절대 습도

```
절대습도(g/m³) = 6.112 × e^(17.67T/(T+243.5)) × RH × 2.1674 / (273.15 + T)
```

### 이슬점 온도

```
γ = ln(RH/100) + 17.62T / (243.12 + T)
이슬점(°C) = 243.12γ / (17.62 - γ)
```

### 보정 계수 적용 (내부 보정)

BME280은 내부 비휘발성 메모리에 보정 계수를 저장하고 라이브러리에서 자동 적용합니다.
사용자는 직접 보정 수식을 구현할 필요가 없습니다.
Bosch 공식 데이터시트 Appendix에 전체 보정 코드가 수록되어 있습니다.

## 주의사항

1. **전원 전압 3.3V**
   - BME280 IC 자체는 최대 3.6V이므로 5V 직접 연결 금지
   - 일부 모듈 보드는 5V 입력 후 내부 LDO로 3.3V 변환 (핀 레이블 확인)

2. **I2C 주소 확인**
   - 같은 버스에 BMP280(0x76/0x77)이 있으면 주소 충돌 가능
   - BME280의 SDO 핀 레벨로 주소 변경

3. **자가 발열**
   - BME280은 연속 측정 시 칩 자체 발열로 온도가 1~2°C 높게 측정될 수 있음
   - Forced Mode로 필요할 때만 측정하면 자가 발열 최소화

4. **센서 오염**
   - 습도 센서가 오염되면 정확도 저하 — 화학 용액, 먼지, 연기 노출 피하기

5. **풀업 저항 (I2C)**
   - I2C 버스에 4.7kΩ 풀업 저항 필요 (SDA, SCL 각각)
   - 대부분의 모듈 보드에 내장되어 있으므로 중복 방지

6. **SPI와 I2C 동시 사용 불가**
   - 한 번에 하나의 인터페이스만 사용 가능

## 캘리브레이션 방법

### 해수면 기압 보정 (고도 정확도 향상)

```cpp
// 현재 위치의 실제 해수면 기압을 기상청에서 확인 후 입력
// 또는: 알려진 고도에서 역산

// 알려진 고도(m)에서 현재 기압으로 해수면 기압 계산
float knownAltitude = 200.0; // m (GPS 또는 지도 확인값)
float measuredPressure = bme.readPressure() / 100.0;
float seaLevelPressure = measuredPressure / pow(1 - knownAltitude / 44330.0, 5.255);
Serial.printf("계산된 해수면 기압: %.2f hPa\n", seaLevelPressure);
```

### 온도 오프셋 보정

```cpp
// 기준 온도계와 비교하여 오프셋 설정
const float TEMP_OFFSET = -1.5; // 자가 발열 보정 등

float correctedTemp = bme.readTemperature() + TEMP_OFFSET;
```

### IIR 필터 설정 (노이즈 감소)

```cpp
bme.setSampling(Adafruit_BME280::MODE_NORMAL,
                Adafruit_BME280::SAMPLING_X16,
                Adafruit_BME280::SAMPLING_X16,
                Adafruit_BME280::SAMPLING_X16,
                Adafruit_BME280::FILTER_X16,   // IIR 필터 계수 16
                Adafruit_BME280::STANDBY_MS_500);
```

## 실사용 팁

- **Forced Mode 권장**: 배터리 프로젝트에서 Normal Mode 대신 Forced Mode로 필요할 때만 측정
- **IIR 필터**: 기압/온도 측정 시 IIR 필터 사용으로 순간 노이즈 제거 — 날씨 모니터링에 권장
- **오버샘플링 vs 속도**: 오버샘플링 X16이 가장 정밀하지만 측정 시간이 길어짐 (약 40ms)
- **고도 정확도**: 기상 조건에 따라 기압이 변하므로 매일 해수면 기압 기준값 업데이트 필요
- **여러 센서 I2C 버스 공유**: 주소가 0x76, 0x77 두 가지뿐이므로 동일 버스에 최대 2개 연결 가능
- **Wi-Fi와 I2C 동작**: ESP32-C3에서 Wi-Fi와 I2C를 동시 사용 시 I2C 타이밍 문제가 발생할 수 있음 — Wire 오류 처리 코드 필수
