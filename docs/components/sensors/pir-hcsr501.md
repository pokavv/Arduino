# PIR 모션 센서 (HC-SR501) — 수동형 적외선 인체 감지

## 개요

HC-SR501은 PIR(Passive Infrared Receiver) 방식의 인체 감지 모션 센서 모듈입니다.
사람이나 동물의 체온에서 방사되는 적외선(원적외선, 8~14µm)의 변화를 감지하여
움직임을 디지털 신호(HIGH/LOW)로 출력합니다.

**측정 원리**
BISS0001 전용 IC와 두 개의 열전 감지 소자로 구성된 프레넬 렌즈(Fresnel Lens)가
시야각 내 적외선 변화를 차동 방식으로 감지합니다.
인체가 없는 정적 환경에서는 두 소자의 출력이 균형을 이루고,
인체가 움직이면 불균형이 발생하여 출력이 HIGH가 됩니다.

**아두이노에서의 활용**
자동 조명 제어, 보안 경보, 에너지 절약 시스템, 방문자 카운터 등에 사용됩니다.
단일 디지털 핀으로 읽어 간단하게 활용 가능합니다.

```

  ┌─────────────────────┐
  │  (가변저항)  (가변저항) │
  │   감도        딜레이   │
  │                      │
  │    ┌──────────┐       │
  │    │ BISS0001 │       │
  │    └──────────┘       │
```

| GND | 접지 (0V) |

## 가변저항 조절

### 감도 조절 (Sensitivity Potentiometer)

```
── CCW(반시계) ← ──────────── → CW(시계) ──
   감지 거리 짧음 (약 3m)       감지 거리 김 (약 7m)
```

### 딜레이 시간 조절 (Time Delay Potentiometer)

```
── CCW(반시계) ← ──────────── → CW(시계) ──
   딜레이 짧음 (약 5초)          딜레이 김 (약 300초)
```

---

## 트리거 모드 (점퍼 설정)

### L 모드 (단일 트리거, Non-Retriggerable)

```
최초 감지 후 딜레이 시간이 지나면 자동으로 LOW 복귀.
딜레이 중 추가 감지가 있어도 딜레이 재시작 안 됨.

타임라인:
  감지 → HIGH ─────────(딜레이)────────→ LOW
  (딜레이 중 재감지 무시)
```

### H 모드 (반복 트리거, Retriggerable) — 기본값

```
감지가 유지되는 동안 딜레이가 계속 연장됨.
마지막 감지 후 딜레이 시간이 경과해야 LOW로 복귀.

타임라인:
  감지 → HIGH ─────(재감지)──────(딜레이)────→ LOW
```

## 통신 프로토콜 — 디지털 출력

HC-SR501은 단순한 디지털 HIGH/LOW 신호를 출력합니다.
특별한 프로토콜 없이 GPIO 디지털 입력으로 읽습니다.

```
출력 상태:
  HIGH (3.3V) = 모션 감지됨
  LOW  (0V)   = 모션 없음 또는 대기 중
```

## 아두이노 연결 방법

### ESP32-C3 연결

```
ESP32-C3          HC-SR501
  5V   ──────────── VCC
  GND  ──────────── GND
  G4   ──────────── OUT  (3.3V 출력, 직결 가능)
```

> OUT 출력이 3.3V이므로 ESP32-C3에 직접 연결 가능합니다.
> VCC는 5V(또는 3.3V) 모두 동작하나, 감지 거리와 안정성은 5V에서 더 좋습니다.

### 기본 코드 예시

```cpp
#define PIR_PIN  4  // OUT 핀 연결 GPIO

void setup() {
  Serial.begin(115200);
  pinMode(PIR_PIN, INPUT);

  Serial.println("PIR 센서 워밍업 중... (60초 대기)");
  delay(60000); // 전원 인가 후 1분 안정화 대기
  Serial.println("준비 완료!");
}

void loop() {
  int motion = digitalRead(PIR_PIN);

  if (motion == HIGH) {
    Serial.println("모션 감지!");
  } else {
    Serial.println("대기 중...");
  }

  delay(500);
}
```

### 인터럽트 방식 (비동기 감지)

```cpp
#define PIR_PIN  4

volatile bool motionDetected = false;

void IRAM_ATTR onMotion() {
  motionDetected = true;
}

void setup() {
  Serial.begin(115200);
  pinMode(PIR_PIN, INPUT);
  attachInterrupt(digitalPinToInterrupt(PIR_PIN), onMotion, RISING);

  Serial.println("PIR 워밍업 60초 대기...");
  delay(60000);
  Serial.println("감지 시작!");
}

void loop() {
  if (motionDetected) {
    motionDetected = false;
    Serial.printf("모션 감지! 시각: %lu ms\n", millis());
    // 여기서 조명 켜기, 알림 전송 등 처리
  }
  // 다른 작업 수행 가능
}
```

### millis() 기반 쿨다운 타이머

```cpp
#define PIR_PIN       4
#define COOLDOWN_MS   3000  // 재감지 억제 시간 (ms)

unsigned long lastTriggerTime = 0;

void loop() {
  if (digitalRead(PIR_PIN) == HIGH) {
    unsigned long now = millis();
    if (now - lastTriggerTime >= COOLDOWN_MS) {
      lastTriggerTime = now;
      Serial.println("모션 감지! (쿨다운 적용)");
      // 동작 수행
    }
  }
}
```

## 주의사항

1. **전원 인가 후 안정화 시간 필수**
   - HC-SR501은 전원 인가 후 **약 60초** 동안 안정화 시간이 필요합니다.
   - 이 시간 동안 OUT 핀이 여러 번 HIGH/LOW를 반복하는 것은 정상입니다.
   - `setup()` 내에서 `delay(60000)` 또는 부트 완료 후 타이머로 처리하세요.

2. **출력 전압 3.3V**
   - OUT 핀은 3.3V를 출력하므로 ESP32-C3 GPIO에 직결 가능합니다.
   - 단, 5V 레벨이 필요한 장치와 연결 시 레벨 업변환 필요.

3. **온도 환경**
   - 주변 온도가 체온(약 36°C)에 가까울수록 감지 정확도가 낮아집니다.
   - 고온 환경(여름 야외, 용광로 근처 등)에서는 오감지 가능.

4. **직사광선/빛 간섭**
   - 프레넬 렌즈는 IR을 투과시키므로 강한 적외선(태양광, 할로겐 조명)이
     직접 닿으면 오감지 발생 가능.

5. **유리 투과 불가**
   - PIR 센서는 일반 유리창을 투과하지 못합니다 (IR 차단).

6. **감지 각도와 설치 높이**
   - 최적 설치 높이: 지면에서 2~2.5m
   - 감지 각도 110° — 모서리보다 중앙 기둥이나 천장 중앙 설치 권장.

7. **2.4초 차단 시간 (H 모드)**
   - H 트리거 모드에서 OUT이 LOW가 된 직후 약 2.4초간 재감지 차단됨.
   - 이 시간 중 움직임이 있어도 HIGH로 전환되지 않습니다.

## 캘리브레이션 방법

HC-SR501은 두 개의 가변저항으로 현장 조정이 가능합니다.

### 감도 최적화

```
1. 가변저항(감도)을 중간 위치로 맞춤
2. 원하는 감지 거리에서 사람이 이동
3. 정상 감지되면 OK; 오감지가 많으면 CCW로 돌려 감도 낮춤
4. 감지 누락이 많으면 CW로 돌려 감도 높임
```

### 딜레이 시간 설정

```
1. 딜레이 가변저항을 최소(CCW)로 설정
2. 원하는 대기 시간에 맞게 조금씩 CW로 돌림
3. 실제 사용 환경에서 테스트하여 최종 위치 결정
```

### 소프트웨어 오감지 필터

```cpp
// 연속 n회 HIGH가 감지될 때만 유효한 모션으로 판단
int confirmCount = 0;
const int CONFIRM_THRESHOLD = 3;

void loop() {
  if (digitalRead(PIR_PIN) == HIGH) {
    confirmCount++;
    if (confirmCount >= CONFIRM_THRESHOLD) {
      Serial.println("확인된 모션 감지!");
      confirmCount = 0;
    }
  } else {
    confirmCount = 0;
  }
  delay(50);
}
```

## 실사용 팁

- **워밍업 중 LED 표시**: 안정화 시간 동안 LED를 점멸하여 사용자에게 안내
- **H 모드 + 딜레이 최소**: 빠른 응답이 필요한 경우 H 모드로 설정하고 딜레이를 최소화 후 소프트웨어로 타이머 관리
- **파워 절약**: PIR 센서는 대기 전류 60µA 미만이므로 배터리 프로젝트에 적합
- **프레넬 렌즈 청소**: 먼지나 거미줄이 렌즈에 쌓이면 감도 저하 — 주기적으로 청소
- **ESP32 딥슬립 연동**: PIR 감지 시 ESP32를 외부 인터럽트로 웨이크업하여 초저전력 시스템 구현 가능
- **렌즈 방향**: 프레넬 렌즈의 홈이 파인 방향이 앞면이므로 설치 시 올바른 방향 확인
