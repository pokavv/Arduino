# NeoPixel / WS2812B 스마트 LED

## 개요

WS2812B(NeoPixel)는 RGB LED와 제어 IC가 하나의 패키지에 통합된 스마트 LED입니다.  
단일 데이터 라인(1-Wire 프로토콜)으로 수십~수백 개의 LED를 직렬 연결하여 각각 개별 제어할 수 있습니다.  
아두이노에서는 LED 스트립 조명, 매트릭스 디스플레이, 애니메이션 조명 효과, 인디케이터 바 등에 활용됩니다.

**NeoPixel**은 Adafruit가 WS2812/WS2812B 호환 제품에 붙인 상표명이며, 동일한 프로토콜을 사용하는  
SK6812, WS2811, APA106 등도 넓은 의미에서 호환됩니다.

## 종류 및 모델 비교

| 모델 | 패키지 | 채널 | 데이터 속도 | 특징 |
|------|--------|------|-----------|------|
| WS2812B | 5050 SMD | RGB | 800kHz | 가장 표준, 내장 IC |
| WS2812 (구형) | 5050 SMD | RGB | 400kHz | WS2812B와 핀 호환, 속도 차이 |
| SK6812 | 5050 SMD | RGBW | 800kHz | 흰색 채널 추가 (4채널) |
| WS2811 | DIP-8 또는 SMD | RGB (외부 LED) | 800kHz | LED와 분리된 드라이버 IC |
| APA106 | 5mm DIP | RGB | 800kHz | 관통형 패키지, 브레드보드 적합 |
| SK6812 Mini-E | 3535 SMD | RGBW | 800kHz | 소형, 역면 납땜 |
| WS2815 | 5050 SMD | RGB | 800kHz | 12V 구동, 단선 복구 기능 |

### WS2812B 제품 형태

| 형태 | 밀도 | 특징 |
|------|------|------|
| LED 스트립 (30/m) | 30 LED/m | 저밀도, 낮은 전력 소비 |
| LED 스트립 (60/m) | 60 LED/m | 표준 밀도, 가장 일반적 |
| LED 스트립 (144/m) | 144 LED/m | 고밀도, 높은 전력 소비 |
| LED 링 | 12, 16, 24개 원형 | 시계/게이지 디스플레이 |
| LED 매트릭스 | 8×8, 16×16 | 픽셀 디스플레이 |
| 개별 모듈 | 1개 | 브레드보드 테스트용 |

## 핀 구성

### WS2812B SMD 5050 패드

```
     VDD ──┐    ┌── DOUT
           │    │
     ┌─────┴────┴─────┐
     │   WS2812B      │
     └─────┬────┬─────┘
           │    │
     VSS ──┘    └── DIN
```

| 핀 | 이름 | 기능 |
|----|------|------|
| VDD | 전원 | 5V 공급 전원 (4.5~5.5V) |
| VSS | GND | 접지 |
| DIN | 데이터 입력 | 이전 LED 또는 마이크로컨트롤러에서 오는 데이터 |
| DOUT | 데이터 출력 | 다음 LED로 전달되는 데이터 (24비트 처리 후 출력) |

### 스트립/모듈 커넥터

| 라벨 | 기능 |
|------|------|
| 5V / VCC | 전원 입력 |
| GND / G | 접지 |
| DIN / D / DATA / IN | 데이터 입력 |
| DOUT / DO / OUT | 데이터 출력 (다음 LED로 연결) |

## 핵심 전기적 스펙

| 파라미터 | 기호 | 값 | 단위 |
|---------|------|-----|------|
| 공급 전압 | VDD | 4.5 ~ 5.5 (typ 5.0) | V |
| 로직 HIGH 입력 전압 | VIH | 0.7 × VDD 이상 | V |
| 로직 LOW 입력 전압 | VIL | 0.3 × VDD 이하 | V |
| 전류 소비 (1개, 최대 밝기) | IMAX | 60 | mA |
| 전류 소비 (1개, 흰색 풀 밝기) | — | 60 | mA |
| 전류 소비 (1개, 빨강 풀 밝기) | — | 20 | mA |
| 전류 소비 (1개, 대기) | ISTANDBY | 1 | mA |
| 데이터 전송 속도 | — | 800 | kHz |
| T0H (0 비트 HIGH 시간) | — | 0.4 ± 150 | µs |
| T0L (0 비트 LOW 시간) | — | 0.85 ± 150 | µs |
| T1H (1 비트 HIGH 시간) | — | 0.8 ± 150 | µs |
| T1L (1 비트 LOW 시간) | — | 0.45 ± 150 | µs |
| 리셋 시간 (TRESET) | — | > 50 | µs |
| 색상 해상도 | — | 24비트 (R8G8B8) | — |
| 동작 온도 | Topr | −25 ~ +85 | °C |
| 저장 온도 | Tstg | −40 ~ +105 | °C |

### 전력 계산

```
전력 소비 = LED 수 × 60mA × 5V (최대 흰색 풀 밝기)
예: 60개 LED 스트립 = 60 × 0.06A × 5V = 18W

실용적인 추정 (평균 밝기):
  LED 수 × 20mA × 5V (혼합 색상 평균)
  예: 60개 = 60 × 0.02A × 5V = 6W
```

## 동작 원리

### 통신 프로토콜 (단선 직렬)

WS2812B는 단일 데이터 라인으로 타이밍 기반 PWM 신호를 사용합니다.

```
비트 '0': HIGH 0.4µs + LOW 0.85µs
비트 '1': HIGH 0.8µs + LOW 0.45µs
리셋:    LOW > 50µs (프레임 종료 신호)
```

### 데이터 프레임 구조

```
[LED1 데이터 24비트][LED2 데이터 24비트]...[RES 50µs 이상]

각 LED 24비트:
  [G7 G6 G5 G4 G3 G2 G1 G0] [R7 R6 R5 R4 R3 R2 R1 R0] [B7 B6 B5 B4 B3 B2 B1 B0]
  MSB 먼저 전송, 순서: G → R → B (주의: RGB 순서 아님!)
```

### 체인 동작

```
MCU → DIN[LED1]DOUT → DIN[LED2]DOUT → DIN[LED3]DOUT → ...
```
- 각 LED는 앞 24비트를 자신의 색상으로 적용하고, 나머지를 DOUT으로 전달합니다.
- 리셋 신호 후 모든 LED가 동시에 업데이트됩니다.

## 아두이노 연결 방법

### 기본 배선

```
ESP32-C3 GPIO ──[470Ω]── DIN (데이터 라인 보호 저항)
5V 전원 ──[100µF 전해 커패시터]── VDD (전원 돌입 전류 방지)
GND ──────────────────────────── VSS
```

> **중요**: 데이터 라인에 직렬 470Ω 저항을 꼭 연결합니다.  
> GPIO와 LED 사이의 임피던스 불일치로 인한 반사파를 억제합니다.

### 레벨 시프터 (3.3V → 5V)

ESP32-C3는 3.3V 출력이지만 WS2812B는 5V 로직을 기대합니다.  
- VIH 최소 = 0.7 × 5V = 3.5V → 3.3V가 부족할 수 있습니다.
- 실제로는 많은 경우 직접 연결해도 동작하지만, **신뢰성을 위해 레벨 시프터 권장**.

```
레벨 시프터 방법 1 — 74AHCT125 IC:
  3.3V 신호 → 74AHCT125 → 5V 신호 → WS2812B

레벨 시프터 방법 2 — MOSFET 레벨 시프터 모듈:
  3.3V LV ←→ 5V HV 양방향 레벨 변환

방법 3 — 다이오드 트릭:
  WS2812B VDD를 4.3V로 낮춤 (5V에 IN4148 직렬 연결)
  → VIH = 0.7 × 4.3V = 3.01V → 3.3V GPIO 신호가 통과
```

### 전원 배선 (장거리/다수 LED)

```
전원 공급기(5V)──┬──── VDD (스트립 시작)
                │     ...중간마다 전원 재주입...
                │──── VDD (스트립 중간)
                └──── GND (스트립 시작 및 마이크로컨트롤러 GND)
```
- 60개 이상 LED는 스트립 중간에도 전원을 추가 연결합니다.
- 스트립 끝 LED의 색상이 틀어진다면 전압 강하 때문입니다.

## 코드 예제 개념

### Adafruit NeoPixel 라이브러리 사용

```cpp
#include <Adafruit_NeoPixel.h>

const int DATA_PIN = 7;       // 데이터 핀
const int NUM_LEDS = 8;       // LED 개수

// 라이브러리 초기화: (LED수, 핀번호, 타입+속도)
Adafruit_NeoPixel strip(NUM_LEDS, DATA_PIN, NEO_GRB + NEO_KHZ800);

void setup() {
    strip.begin();            // 초기화
    strip.setBrightness(50);  // 전체 밝기 (0~255), 전력 절약을 위해 낮게 설정
    strip.show();             // 초기 상태 전송 (모두 꺼짐)
}

void loop() {
    // 특정 LED 색상 설정 (인덱스, R, G, B)
    strip.setPixelColor(0, strip.Color(255, 0, 0));    // 첫 번째 = 빨강
    strip.setPixelColor(1, strip.Color(0, 255, 0));    // 두 번째 = 녹색
    strip.setPixelColor(2, strip.Color(0, 0, 255));    // 세 번째 = 파랑
    strip.show();  // 실제로 LED에 출력 (show() 전까지는 버퍼에만 저장)
    delay(1000);

    // 전체 색상 설정
    for (int i = 0; i < NUM_LEDS; i++) {
        strip.setPixelColor(i, strip.Color(255, 100, 0)); // 주황
    }
    strip.show();
    delay(1000);

    strip.clear(); // 전체 끄기
    strip.show();
    delay(500);
}
```

### FastLED 라이브러리 사용 (고급)

```cpp
#include <FastLED.h>

const int DATA_PIN = 7;
const int NUM_LEDS = 8;

CRGB leds[NUM_LEDS];

void setup() {
    FastLED.addLeds<WS2812B, DATA_PIN, GRB>(leds, NUM_LEDS);
    FastLED.setBrightness(50);
}

void loop() {
    // CRGB 색상 직접 할당
    leds[0] = CRGB::Red;
    leds[1] = CRGB(0, 255, 0);     // G,R,B 순서 주의
    leds[2] = CHSV(160, 255, 255); // HSV 색상 (파랑 계열)
    FastLED.show();
    delay(1000);

    // 무지개 효과
    for (int hue = 0; hue < 256; hue++) {
        fill_rainbow(leds, NUM_LEDS, hue, 10);
        FastLED.show();
        delay(20);
    }
}
```

## 마킹/식별 방법

| 확인 요소 | 설명 |
|---------|------|
| 5050 패키지 각인 | 모서리 중 하나에 작은 삼각형 또는 점 = GND |
| PCB 실크스크린 | DIN, DOUT, 5V, GND 라벨 확인 |
| 데이터 흐름 방향 | 스트립에 화살표(▶) 또는 삼각형 표시 방향이 데이터 흐름 |
| 커넥터 색상 (스트립) | 빨강=5V, 흰색/녹색=데이터, 검정=GND |

## 주의사항

1. **전원 분리 공급**: 많은 LED(> 10개)는 아두이노 5V 핀이 아닌 별도 5V 전원 어댑터 사용 권장. USB 허브 전류 초과 방지.
2. **전원 커패시터 필수**: 전원 입력 라인에 100~1000µF 전해 커패시터 연결. LED 온/오프 시 순간 전류 급변을 완충.
3. **데이터 저항**: DIN 라인에 300~500Ω 저항을 GPIO와 LED 사이에 직렬 삽입. 고주파 반사 억제.
4. **GND 공통**: 마이크로컨트롤러 GND와 LED 전원 GND를 반드시 공통 연결.
5. **밝기 제한**: 테스트 시 setBrightness(50) 이하로 제한. 풀 밝기는 엄청난 전류를 소비하며 과열 위험.
6. **타이밍 민감**: WS2812B는 타이밍에 매우 민감합니다. 인터럽트가 많은 환경(Wi-Fi 동작 중)에서 색상 깜빡임이 발생할 수 있음 → DMA 기반 라이브러리 사용 권장.
7. **ESP32 RMT 활용**: ESP32 계열은 RMT(Remote Control) 하드웨어를 사용하여 CPU 독립적으로 WS2812B 신호를 생성합니다. FastLED/Adafruit 라이브러리가 자동 지원.
8. **ESD 방지**: 정전기로 인한 내부 IC 손상에 주의.

## 자주 쓰이는 회로 패턴

### 패턴 1: 기본 소수 LED 연결 (USB 전원)

```
ESP32 GPIO7 ──[470Ω]── DIN
ESP32 5V ────[100µF]── 5V
ESP32 GND ──────────── GND
```
- 10개 이하 LED, USB 전원으로 충분할 때.

### 패턴 2: 다수 LED + 외부 전원

```
외부 5V PSU ──┬──[100µF]── LED 스트립 5V
              └──────────── ESP32 5V (선택적, 전압이 동일할 때)
외부 GND ──────┬──────────── LED 스트립 GND
               └──────────── ESP32 GND (공통 GND 필수)
ESP32 GPIO7 ──[470Ω]───────── DIN
```

### 패턴 3: 레벨 시프터 포함 (신뢰성 우선)

```
ESP32 GPIO ──[470Ω]── LV (레벨시프터) HV ──[470Ω]── DIN
ESP32 3.3V ────────── LV VCC
5V ─────────────────── HV VCC
GND ────────────────── GND
```

### 패턴 4: 상태 표시 매핑

```
Wi-Fi 연결 중  → 파랑 숨쉬기 효과
Wi-Fi 연결됨   → 초록 고정
MQTT 수신      → 흰색 플래시
오류 상태      → 빨강 빠른 점멸
절전 모드      → 모두 꺼짐
```
