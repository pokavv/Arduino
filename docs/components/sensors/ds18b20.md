# DS18B20 — 1-Wire 디지털 온도 센서

## 개요

DS18B20은 Maxim Integrated(현 Analog Devices)에서 제조한 디지털 온도 센서로,
Dallas Semiconductor의 **1-Wire** 프로토콜을 사용하여 단 하나의 데이터 핀으로 통신합니다.
내부에 온도 센싱 소자, ADC, EEPROM, 64비트 고유 ROM 코드를 내장합니다.

**측정 원리**
온도에 따라 발진 주파수가 변하는 발진기와 기준 발진기를 비교하는 방식으로 온도를 디지털 변환합니다.
분해능은 9 ~ 12비트로 프로그래밍 가능하며, 12비트 시 0.0625°C 단위로 측정합니다.

**아두이노에서의 활용**
하나의 GPIO 핀에 최대 수십 개의 DS18B20을 병렬 연결(기생 전원 모드 포함)할 수 있어
온도 분포 모니터링, 수조 온도 관리, 파이프 온도 감시 등에 널리 사용됩니다.
방수형 스테인리스 프로브 제품이 있어 액체 온도 측정에 매우 적합합니다.

## 핀 구성

### TO-92 패키지 (평면이 앞을 향할 때)

```
    ┌─────┐
    │ ●●● │  ← 반원형 돌출면이 앞
    └──┬──┘
       │
  1   2   3
 GND DATA VDD
```

| 핀 번호 | 이름 | 역할 |
|---------|------|------|
| 1 | GND | 접지 (0V) |
| 2 | DQ (DATA) | 1-Wire 데이터 입출력 (오픈 드레인) |
| 3 | VDD | 전원 (+3.0 ~ 5.5V), 기생 모드 시 GND 연결 |

> **주의**: TO-92 패키지는 제조사에 따라 핀 배열이 다를 수 있습니다.
> 반드시 구매한 제품의 데이터시트를 확인하세요. DS18B20 정품과 호환품의 핀 배열이 다른 경우가 있습니다.

### 방수 프로브 (3선 케이블)

| 선 색상 | 역할 |
|---------|------|
| 빨간색 | VDD (+전원) |
| 검은색 | GND (접지) |
| 노란색/흰색 | DQ (데이터) |

## 통신 프로토콜 — 1-Wire

### 1-Wire 개요

1-Wire는 Dallas Semiconductor가 개발한 단선 직렬 통신 프로토콜로,
전원(기생 모드), 데이터, 클록을 단 하나의 신호선으로 처리합니다.
마스터(MCU)가 버스를 제어하고 슬레이브(DS18B20)가 응답합니다.

### 기본 타이밍

```
리셋 펄스:
  마스터가 480µs 이상 LOW 출력
  → 마스터 HIGH 복귀 (풀업 저항 역할)
  → 슬레이브가 60~240µs 후 LOW(존재 펄스) 출력
  → 마스터가 존재 펄스 감지

비트 쓰기:
  '1' 쓰기: 1~15µs LOW 후 슬롯 끝까지 HIGH
  '0' 쓰기: 60~120µs LOW 유지

비트 읽기:
  마스터가 1~15µs LOW 후 HIGH 복귀
  → 슬레이브가 데이터에 따라 LOW 또는 HIGH 유지
  → 마스터가 15µs 이내에 버스 레벨 샘플링
```

### ROM 명령어

| 명령어 | 코드 | 설명 |
|--------|------|------|
| Search ROM | 0xF0 | 버스의 모든 장치 검색 |
| Read ROM | 0x33 | 단일 장치의 64비트 ROM 읽기 |
| Match ROM | 0x55 | 특정 ROM 주소 장치 선택 |
| Skip ROM | 0xCC | ROM 주소 무시 (단일 장치 버스에서 사용) |
| Alarm Search | 0xEC | 알람 조건 장치만 검색 |

### 기능 명령어

| 명령어 | 코드 | 설명 |
|--------|------|------|
| Convert T | 0x44 | 온도 변환 시작 |
| Write Scratchpad | 0x4E | TH, TL, Config 레지스터 쓰기 |
| Read Scratchpad | 0xBE | 스크래치패드 9바이트 읽기 |
| Copy Scratchpad | 0x48 | TH, TL, Config를 EEPROM에 저장 |
| Recall E² | 0xB8 | EEPROM에서 TH, TL, Config 복원 |
| Read Power Supply | 0xB4 | 전원 공급 방식 확인 |

### 스크래치패드 구조 (9바이트)

```
Byte 0: Temperature LSB
Byte 1: Temperature MSB
Byte 2: TH Register (상한 알람)
Byte 3: TL Register (하한 알람)
Byte 4: Configuration Register (분해능)
Byte 5: Reserved (0xFF)
Byte 6: Reserved
Byte 7: Reserved (0x10)
Byte 8: CRC
```

### Configuration Register (분해능 설정)

```
비트 [6:5] R1:R0 → 분해능
  00 = 9비트  (0.5°C,    93.75ms)
  01 = 10비트 (0.25°C,  187.5ms)
  10 = 11비트 (0.125°C,  375ms)
  11 = 12비트 (0.0625°C, 750ms)  ← 기본값
```

## 아두이노 연결 방법

### 표준 모드 (외부 전원)

```
ESP32-C3          DS18B20
  3.3V ──────────── VDD (3번)
  GND  ──────────── GND (1번)
  G4   ──┬───────── DQ  (2번)
         │
        4.7kΩ (풀업 저항 필수)
         │
  3.3V ──┘
```

### 기생 전원 모드 (Parasite Power)

```
  ESP32-C3          DS18B20
    3.3V ─┬──────── VDD (3번) — VDD와 GND 모두 연결
    GND  ─┤──────── GND (1번)
    G4   ─┼─┬────── DQ  (2번)
           │ │
          4.7kΩ
           │ │
    3.3V ──┘ │
             │  ← DQ에서 기생 전원 공급
```

> **기생 전원 모드**: VDD 핀을 GND에 연결하고 DQ 핀에서 전원 공급.
> 배선 절약에 유리하지만, 온도 변환 중 충분한 전류 공급을 위해
> 변환 명령 직후 DQ를 강제 HIGH로 유지해야 합니다 (`parasite` 옵션).

### 다중 센서 연결

```
  ESP32-C3           DS18B20 #1   DS18B20 #2   DS18B20 #3
    3.3V ─────────┬─── VDD    ──── VDD    ──── VDD
    GND  ─────────┼─── GND    ──── GND    ──── GND
    G4   ──┬──────┼─── DQ     ──── DQ     ──── DQ
           │      └─── (버스 공유)
          4.7kΩ
           │
    3.3V ──┘
```

64비트 고유 ROM 코드로 각 센서를 개별 제어합니다.

### 라이브러리 설치

Arduino IDE > 라이브러리 매니저에서:
- `OneWire` by Jim Studt 설치
- `DallasTemperature` by Miles Burton 설치

### 기본 코드 예시

```cpp
#include <OneWire.h>
#include <DallasTemperature.h>

#define ONE_WIRE_PIN  4  // DQ 연결 GPIO

OneWire           oneWire(ONE_WIRE_PIN);
DallasTemperature sensors(&oneWire);

void setup() {
  Serial.begin(115200);
  sensors.begin();

  // 연결된 센서 수 출력
  Serial.printf("발견된 DS18B20 수: %d\n", sensors.getDeviceCount());

  // 12비트 분해능 설정 (기본값이지만 명시적으로 설정)
  sensors.setResolution(12);
}

void loop() {
  sensors.requestTemperatures(); // 온도 변환 명령 (750ms 소요)

  // 첫 번째 센서 값 읽기
  float tempC = sensors.getTempCByIndex(0);

  if (tempC == DEVICE_DISCONNECTED_C) {
    Serial.println("센서 연결 오류!");
    return;
  }

  Serial.printf("온도: %.4f °C\n", tempC); // 12비트: 0.0625°C 단위
  delay(1000);
}
```

### 다중 센서 주소 기반 읽기

```cpp
DeviceAddress sensor1, sensor2;

void setup() {
  sensors.begin();
  sensors.getAddress(sensor1, 0); // 인덱스 0번 주소 읽기
  sensors.getAddress(sensor2, 1); // 인덱스 1번 주소 읽기
}

void loop() {
  sensors.requestTemperatures();
  float t1 = sensors.getTempC(sensor1);
  float t2 = sensors.getTempC(sensor2);
  Serial.printf("센서1: %.2f°C  센서2: %.2f°C\n", t1, t2);
  delay(1000);
}
```

## 측정값 계산 공식

### 온도 변환 (원시 데이터 → °C)

스크래치패드 Byte0(LSB)와 Byte1(MSB)에서 16비트 값을 조합:

```
rawTemp = (MSB << 8) | LSB

양수: temperature = rawTemp × 0.0625
음수: temperature = (rawTemp - 65536) × 0.0625  (2의 보수)
```

### 비트별 온도 값

```
12비트 기준 LSB 구조:
  비트 15~11: 부호 (S)
  비트 10~4 : 정수부 (7비트)
  비트 3~0  : 소수부 (4비트 → 2⁻¹, 2⁻², 2⁻³, 2⁻⁴)

예: raw = 0x0550 (1360 decimal)
  temperature = 1360 × 0.0625 = 85.0°C

예: raw = 0xFC90 (음수 예시)
  temperature = (0xFC90 - 0x10000) × 0.0625 = -55.0°C
```

## 주의사항

1. **풀업 저항 필수**
   - 4.7kΩ 풀업 저항 없이는 통신 불가
   - 버스 길이가 길거나 센서 수가 많으면 2.2kΩ으로 낮춤

2. **변환 완료 대기**
   - 12비트 분해능에서 변환 완료까지 최대 **750ms** 소요
   - `requestTemperatures()` 호출 후 충분히 대기
   - 비동기 방식: `sensors.setWaitForConversion(false)` 후 750ms 후 읽기

3. **기생 전원 모드 제한**
   - 기생 모드에서는 12비트 변환 중 충분한 전류 공급 필요
   - 변환 중 DQ 핀을 강제 HIGH (Active HIGH) 유지 필요
   - 온도 >100°C 측정 시 기생 모드 사용 불가

4. **핀 배열 확인**
   - 중국산 호환품(예: DS18B20 클론)은 정품과 핀 배열이 반대인 경우가 있습니다
   - 잘못 연결 시 과열 및 파손 가능 → 연결 전 정확히 확인

5. **방수 프로브 케이블 색상**
   - 케이블 색상이 제조사마다 다를 수 있으므로 멀티미터로 확인 권장

## 캘리브레이션 방법

DS18B20은 ±0.5°C 공장 캘리브레이션 완료 제품입니다.
정밀 적용 시 소프트웨어 오프셋 보정을 사용합니다.

### TH/TL 알람 레지스터 활용

```cpp
// 스크래치패드에 온도 알람 임계값 저장 (EEPROM 보존 가능)
sensors.setHighAlarmTemp(sensor1, 30);  // 30°C 이상 알람
sensors.setLowAlarmTemp(sensor1, 10);   // 10°C 이하 알람

// 알람 발생 센서 스캔
sensors.requestTemperatures();
DeviceAddress alarmDevice;
oneWire.reset_search();
while (sensors.alarmSearch(alarmDevice)) {
  Serial.println("알람 센서 발견!");
}
```

### 소프트웨어 오프셋 보정

```cpp
const float OFFSET = -0.3; // 기준 온도계 비교 후 결정
float corrected = sensors.getTempCByIndex(0) + OFFSET;
```

## 실사용 팁

- **비동기 읽기로 750ms 블로킹 방지**: `setWaitForConversion(false)` 설정 후 millis()로 750ms 경과 확인 후 읽기
- **ROM 주소 출력 루틴**: 센서 교체 시 새 주소를 확인하는 `printAddress()` 유틸 함수를 코드에 포함
- **방수 프로브 선택**: 수조, 발효기 등 액체 접촉 환경에는 반드시 방수 프로브 모델 사용
- **EMI 환경 대응**: 긴 케이블 사용 시 DQ 라인에 100Ω 직렬 저항 + 100pF 캐패시터 추가로 노이즈 억제
- **전원 디커플링**: VDD와 GND 사이에 100nF 커패시터 배치
- **정품 확인**: 시장에 클론(DS18B20S 등)이 많으므로 신뢰할 수 있는 공급처에서 구매 권장
