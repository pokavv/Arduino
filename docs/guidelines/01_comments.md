# 주석 가이드

---

## 주석의 원칙

좋은 주석은 **왜(why)** 와 **어떻게 동작하는지(how it works)** 를 설명합니다.
코드를 읽으면 무엇을 하는지는 알 수 있지만, 왜 그렇게 했는지는 알 수 없기 때문입니다.

특히 이 프로젝트는 하드웨어 + 코드를 처음 배우는 사람을 대상으로 하므로
**하드웨어 개념, 부품 특성, 연결 방법**도 주석에 포함합니다.

---

## 파일 헤더 주석 형식

모든 `.ino` 파일 최상단에 아래 형식으로 작성합니다.
초보자도 파일만 열면 무엇을 해야 하는지 알 수 있도록 작성합니다.

```cpp
/*
 * X-XX 기능 이름 — 한 줄 설명
 * ================================================================
 *
 * [핵심 개념 설명]
 *   이 예제에서 처음 나오는 하드웨어 또는 코드 개념을 설명.
 *   용어, 동작 원리, 주의사항 포함.
 *
 * [준비물]
 *   필요한 부품 목록 (없으면 "없음 — 보드만으로 테스트 가능" 명시)
 *
 * [연결 방법]
 *   핀 이름 → 부품 이름 → 다음 연결 형식으로 작성.
 *   예: G2 → 저항(220Ω) → LED 긴 다리(+) → LED 짧은 다리(-) → GND
 *   그림(ASCII)으로 표현할 수 있으면 더 좋다.
 */
```

### 실제 예시 — 간단한 예제

```cpp
/*
 * 1-01 디지털 출력 — LED 켜고 끄기
 * ================================================================
 *
 * [하드웨어 기초 용어]
 *
 *  GND (접지)
 *    전기의 기준점. 배터리의 - 극과 같은 개념.
 *    모든 부품은 GND가 연결되어야 전기가 흐른다.
 *
 *  HIGH / LOW
 *    HIGH = 전기 있음 = 3.3V
 *    LOW  = 전기 없음 = 0V
 *
 *  저항 (220Ω)
 *    전류를 줄여주는 부품. LED에 안 달면 LED가 탄다.
 *
 *  LED
 *    긴 다리(+) → 전기 들어오는 쪽
 *    짧은 다리(-) → GND 쪽. 방향 바꾸면 안 켜진다.
 *
 * [준비물]
 *   LED 1개, 220Ω 저항 1개
 *
 * [연결 방법]
 *   G2 → 저항(220Ω) → LED 긴 다리(+) → LED 짧은 다리(-) → GND
 */
```

### 실제 예시 — 개념 설명이 필요한 예제

```cpp
/*
 * 1-06 인터럽트 — 버튼 누르는 순간 즉시 감지 (FALLING)
 * ================================================================
 *
 * [인터럽트란?]
 *   일반적으로 코드는 loop() 안에서 위→아래 순서대로 실행된다.
 *   delay(1000)이 있으면 1초 동안 다른 걸 아무것도 못 한다.
 *
 *   인터럽트는 "어떤 신호가 오는 순간 즉시 특정 함수를 실행"하는 기능.
 *   delay 중에도, 다른 코드 실행 중에도 즉시 반응한다.
 *
 * [FALLING이란?]
 *   신호가 HIGH(3.3V)에서 LOW(0V)로 "떨어지는" 순간.
 *   BOOT 버튼을 누르면 G9이 HIGH→LOW로 바뀐다 → FALLING 발생.
 *
 * [인터럽트 함수 작성 규칙]
 *   1) 함수 앞에 IRAM_ATTR 붙이기 — 빠른 메모리에 올려두는 것
 *   2) 함수 내용은 최대한 짧게 — 신호(플래그)만 남기고 즉시 끝냄
 *   3) 공유 변수 앞에 volatile 붙이기
 *
 * [준비물]
 *   없음 — 보드의 BOOT 버튼(G9) 사용
 */
```

## 인라인 주석

코드 오른쪽에 짧게 씁니다. 하드웨어 특성, 단위, 범위를 명시합니다.

```cpp
pinMode(BUILTIN_LED_PIN, OUTPUT);
digitalWrite(BUILTIN_LED_PIN, HIGH);   // HIGH = 꺼짐 (내장 LED는 Active LOW — 반대!)

int raw = analogRead(ADC_PIN);         // 0~4095 (12비트 ADC)
float voltage = raw * (3.3f / 4095.0f);  // 0.0~3.3V 범위로 변환

volatile bool buttonPressed = false;   // volatile: 인터럽트와 loop() 공유 변수 필수
```

## 하드웨어 관련 필수 주석 항목

하드웨어와 연관된 코드에는 아래 정보를 주석으로 남깁니다.

| 항목 | 예시 |
|------|------|
| Active LOW 부품 | `// LOW = 켜짐, HIGH = 꺼짐 (Active LOW)` |
| 핀 최대 전압 | `// 최대 3.3V — 5V 직접 연결 금지` |
| 전류 제한 | `// GPIO 핀 최대 40mA — 대전류 부하는 트랜지스터 필요` |
| ADC 단위 | `// 0~4095 (12비트), Wi-Fi 켜면 불안정 가능` |
| PWM 특이사항 | `// analogWrite() 안 됨 → ledcWrite() 사용` |
| 부팅핀 주의 | `// G0, G9은 부팅핀 — 부팅 시 로직 주의` |
| I2C 주소 | `// SSD1306 기본 주소 0x3C (일부 모듈 0x3D)` |

## 좋은 주석 vs 나쁜 주석

```cpp
// ❌ 나쁜 주석 — 코드를 그대로 번역
digitalWrite(8, LOW);  // 8번 핀을 LOW로 설정

// ✅ 좋은 주석 — 이유 + 하드웨어 특성 설명
digitalWrite(8, LOW);  // 내장 LED 켜기 (G8은 Active LOW — LOW가 켜지는 신호)
```

```cpp
// ❌ 나쁜 주석 — 명백한 내용
i++;  // i를 1 증가

// ✅ 좋은 주석 — 이유 설명
i++;  // 다음 센서 채널로 이동
```

```cpp
// ❌ 나쁜 주석 — 코드와 불일치 (거짓말하는 주석)
// 30도 이상이면 팬 켜기
if (temp > 40) { ... }

// ✅ 최신 상태 유지
// 40도 이상에서 팬 켜기 (30→40으로 변경: 너무 잦은 켜짐 방지)
if (temp > 40) { ... }
```
