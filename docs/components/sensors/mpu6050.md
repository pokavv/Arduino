# MPU-6050 — 6축 가속도 / 자이로 센서 (IMU)

## 개요

MPU-6050은 InvenSense(현 TDK)에서 제조한 6축 관성 측정 장치(IMU: Inertial Measurement Unit)입니다.
3축 가속도계(Accelerometer)와 3축 자이로스코프(Gyroscope)를 하나의 칩에 통합하고,
내장 DMP(Digital Motion Processor)가 센서 퓨전(Sensor Fusion)을 처리합니다.

**측정 원리**
- 가속도: MEMS 커패시터 방식 — 미세 질량 추(seismic mass)의 변위를 정전용량으로 검출
- 자이로: MEMS 코리올리(Coriolis) 효과 — 진동 중인 질량에 가해지는 코리올리 힘을 측정
- DMP: 가속도+자이로 데이터를 융합하여 쿼터니온(Quaternion) 및 오일러 각도(Euler Angles) 출력

**아두이노에서의 활용**
드론 자세 제어, 손떨림 보정, 보행 분석, 제스처 인식, 자율주행 로봇 기울기 감지 등에 사용됩니다.

## 통신 프로토콜 — I2C

### I2C 주소

```
AD0 핀:
  LOW  (GND) → 0x68  ← 기본값 (GY-521 모듈 기본)
  HIGH (VCC) → 0x69
```

### 주요 레지스터

| 주소 | 이름 | 설명 |
|------|------|------|
| 0x1A | CONFIG | DLPF (디지털 저역통과 필터) 설정 |
| 0x1B | GYRO_CONFIG | 자이로 풀스케일 범위 |
| 0x1C | ACCEL_CONFIG | 가속도 풀스케일 범위 |
| 0x3B | ACCEL_XOUT_H | 가속도 X축 상위 바이트 (6바이트: ACCEL X Y Z) |
| 0x41 | TEMP_OUT_H | 온도 상위 바이트 (2바이트) |
| 0x43 | GYRO_XOUT_H | 자이로 X축 상위 바이트 (6바이트: GYRO X Y Z) |
| 0x6B | PWR_MGMT_1 | 전원 관리 (슬립 모드, 클록 소스) |
| 0x75 | WHO_AM_I | 칩 ID (0x68) |

### 초기화 순서

```
1. WHO_AM_I 레지스터 읽기 (0x68 확인)
2. PWR_MGMT_1 = 0x00 (슬립 해제, 클록 활성화)
3. SMPLRT_DIV = 샘플링 분주비 설정
4. CONFIG = DLPF 설정
5. GYRO_CONFIG = 풀스케일 범위 설정
6. ACCEL_CONFIG = 가속도 풀스케일 범위 설정
```

### 기본 코드 예시 (Adafruit)

```cpp
#include <Adafruit_MPU6050.h>
#include <Adafruit_Sensor.h>
#include <Wire.h>

Adafruit_MPU6050 mpu;

void setup() {
  Serial.begin(115200);
  Wire.begin(5, 6); // SDA=G5, SCL=G6

  if (!mpu.begin()) {
    Serial.println("MPU6050을 찾을 수 없습니다!");
    while (1) delay(10);
  }

  mpu.setAccelerometerRange(MPU6050_RANGE_8_G);
  mpu.setGyroRange(MPU6050_RANGE_500_DEG);
  mpu.setFilterBandwidth(MPU6050_BAND_21_HZ);
  Serial.println("MPU6050 초기화 완료");
}

void loop() {
  sensors_event_t accel, gyro, temp;
  mpu.getEvent(&accel, &gyro, &temp);

  Serial.printf("가속도 X:%.2f Y:%.2f Z:%.2f m/s²\n",
    accel.acceleration.x, accel.acceleration.y, accel.acceleration.z);
  Serial.printf("자이로  X:%.2f Y:%.2f Z:%.2f rad/s\n",
    gyro.gyro.x, gyro.gyro.y, gyro.gyro.z);
  Serial.printf("온도:   %.2f °C\n", temp.temperature);

  delay(500);
}
```

### 레지스터 직접 제어 코드

```cpp
#include <Wire.h>

const uint8_t MPU_ADDR    = 0x68;
const uint8_t PWR_MGMT_1  = 0x6B;
const uint8_t ACCEL_XOUT_H= 0x3B;
const uint8_t GYRO_XOUT_H = 0x43;

struct ImuData {
  int16_t ax, ay, az;
  int16_t gx, gy, gz;
};

void mpuInit() {
  Wire.beginTransmission(MPU_ADDR);
  Wire.write(PWR_MGMT_1);
  Wire.write(0x00); // 슬립 해제
  Wire.endTransmission();
}

ImuData mpuRead() {
  Wire.beginTransmission(MPU_ADDR);
  Wire.write(ACCEL_XOUT_H);
  Wire.endTransmission(false);
  Wire.requestFrom(MPU_ADDR, (uint8_t)14); // ACCEL(6) + TEMP(2) + GYRO(6)

  ImuData d;
  d.ax = (Wire.read() << 8) | Wire.read();
  d.ay = (Wire.read() << 8) | Wire.read();
  d.az = (Wire.read() << 8) | Wire.read();
  Wire.read(); Wire.read(); // 온도 (무시)
  d.gx = (Wire.read() << 8) | Wire.read();
  d.gy = (Wire.read() << 8) | Wire.read();
  d.gz = (Wire.read() << 8) | Wire.read();
  return d;
}
```

## 측정값 계산 공식

### 가속도 (m/s² 또는 g)

```
가속도(g) = raw_accel / 감도 (LSB/g)

예: ±2g 설정, raw_ax = 8192
  ax = 8192 / 16384 = 0.5 g = 4.905 m/s²

가속도(m/s²) = 가속도(g) × 9.80665
```

### 자이로 (°/s 또는 rad/s)

```
각속도(°/s) = raw_gyro / 감도 (LSB/°/s)

예: ±250°/s 설정, raw_gx = 1310
  gx = 1310 / 131 = 10.0 °/s

각속도(rad/s) = 각속도(°/s) × π / 180
```

### 온도 (°C)

```
온도(°C) = raw_temp / 340 + 36.53
```

### 기울기 각도 (가속도만 이용, 정적 상태에서만 유효)

```
roll  (X축 기울기) = atan2(ay, az) × 180 / π
pitch (Y축 기울기) = atan2(-ax, sqrt(ay² + az²)) × 180 / π
yaw   (Z축 회전)   = 자이로 적분으로만 계산 가능 (가속도 불가)
```

### 상보 필터 (Complementary Filter) — 자이로 드리프트 보정

```cpp
float alpha = 0.96; // 자이로 가중치 (0.9 ~ 0.98 권장)
float dt    = 0.01; // 루프 주기 (초)

float rollFilt  = alpha * (rollFilt  + gx * dt) + (1-alpha) * accelRoll;
float pitchFilt = alpha * (pitchFilt + gy * dt) + (1-alpha) * accelPitch;
```

## 주의사항

1. **전원 전압 주의**
   - MPU-6050 IC는 최대 3.46V — 5V 직접 연결 금지
   - GY-521 모듈은 온보드 레귤레이터가 있어 5V 허용 (모듈 핀 레이블 확인)

2. **자이로 드리프트**
   - 자이로스코프는 각속도를 적분하여 각도를 계산하므로, 시간이 지남에 따라 오차 누적
   - 장시간 사용 시 가속도계 데이터와 융합(상보 필터, 칼만 필터) 필수

3. **진동 환경**
   - 강한 진동(모터, 드릴 등) 환경에서 가속도계 측정값이 왜곡됨
   - DLPF(디지털 저역통과 필터) 설정으로 고주파 노이즈 제거

4. **I2C 풀업**
   - SDA, SCL에 4.7kΩ 풀업 저항 필요 (모듈 내장 여부 확인)

5. **INT 핀 처리**
   - 인터럽트를 사용하지 않더라도 INT 핀이 플로팅 상태면 오작동 가능
   - 미사용 시 10kΩ으로 GND에 풀다운 권장

6. **칩 방향**
   - 축 방향은 패키지의 점(dot) 또는 데이터시트의 축 그림 확인
   - 기판에 설치할 때 방향에 따라 소프트웨어에서 축 부호 반전 필요

## 캘리브레이션 방법

### 정적 오프셋 캘리브레이션

```cpp
// 1. 센서를 수평으로 놓고 정지 상태에서 1000회 샘플 평균
long axSum = 0, aySum = 0, azSum = 0;
long gxSum = 0, gySum = 0, gzSum = 0;
const int SAMPLES = 1000;

for (int i = 0; i < SAMPLES; i++) {
  ImuData d = mpuRead();
  axSum += d.ax; aySum += d.ay; azSum += d.az;
  gxSum += d.gx; gySum += d.gy; gzSum += d.gz;
  delay(3);
}

// 오프셋 계산 (정적 상태에서 Z축 가속도 = +1g = 16384)
int16_t axOffset = axSum / SAMPLES;
int16_t ayOffset = aySum / SAMPLES;
int16_t azOffset = azSum / SAMPLES - 16384; // 중력 빼기
int16_t gxOffset = gxSum / SAMPLES;
int16_t gyOffset = gySum / SAMPLES;
int16_t gzOffset = gzSum / SAMPLES;

// 이후 측정값에서 오프셋 빼기
// ax_cal = raw_ax - axOffset; ...
```

### 하드웨어 오프셋 레지스터 사용

```cpp
// MPU-6050 내장 오프셋 레지스터 직접 설정
mpu.setXAccelOffset(axOffset);
mpu.setYAccelOffset(ayOffset);
mpu.setZAccelOffset(azOffset);
mpu.setXGyroOffset(gxOffset);
mpu.setYGyroOffset(gyOffset);
mpu.setZGyroOffset(gzOffset);
```

## 실사용 팁

- **칼만 필터 vs 상보 필터**: 상보 필터는 구현이 간단하고 리소스가 적으며, 드론·로봇에 충분히 실용적
- **DMP 활용**: MPU-6050의 DMP를 활성화하면 칩 내부에서 쿼터니온을 계산하여 MCU 부하 감소
- **샘플링 주파수**: 드론 자세 제어는 100~200Hz, 보행 분석은 50Hz, 제스처는 10~20Hz가 일반적
- **I2C 클록**: 기본 100kHz보다 400kHz(Fast Mode)로 설정하면 데이터 전송 속도 향상
- **자이로 전용 단기 측정**: 빠른 동작(스윙, 충격)만 감지한다면 가속도계 비활성화로 소비 전류 절감
- **진동 절연**: PCB 아래에 방진 패드를 두면 모터 진동이 자이로 측정에 미치는 영향 감소
