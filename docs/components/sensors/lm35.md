# LM35 — 아날로그 온도 센서

## 개요

LM35는 Texas Instruments에서 제조한 정밀 아날로그 온도 센서 IC입니다.
측정 온도에 비례하는 아날로그 전압을 출력하며, **10 mV/°C** 선형 특성을 가집니다.
외부 캘리브레이션 없이 사용 가능하며, 섭씨 스케일로 직접 출력됩니다.

**측정 원리**
밴드갭(Bandgap) 기준 전압 회로와 내부 증폭기를 사용하여 온도에 비례하는 전압을 생성합니다.
온도가 1°C 상승할 때마다 출력 전압이 정확히 10 mV 상승합니다.

**아두이노에서의 활용**
ADC로 출력 전압을 읽어 온도로 변환합니다.
구조가 단순하고 회로가 간단하여 입문자 학습 프로젝트에 적합합니다.
응답 속도가 빠르고 낮은 전류 소비로 배터리 프로젝트에도 활용됩니다.

## 통신 프로토콜 — 아날로그 전압 출력

LM35는 디지털 프로토콜 없이 순수 아날로그 전압을 출력합니다.
MCU의 ADC로 전압을 읽어 온도로 환산합니다.

### 출력 전압 특성

```
Vout = 10 mV × T(°C)

예시:
  T = 0°C   → Vout = 0 V
  T = 25°C  → Vout = 0.250 V (250 mV)
  T = 100°C → Vout = 1.000 V (1000 mV)
  T = 150°C → Vout = 1.500 V (1500 mV)
```

## 아두이노 연결 방법

### 기본 연결 (0°C 이상만 측정)

```
ESP32-C3          LM35
  3.3V ──────────── VS  (1번)
  GND  ──────────── GND (3번)
  G1   ──────────── Vout(2번)  → ADC 입력
```

> **ESP32-C3 주의**: ADC 입력 범위가 0 ~ 3.3V이므로 LM35를 3.3V로 공급하면
> 최대 측정 온도는 330°C이지만 LM35 자체 최대치인 150°C가 실질 한계입니다.

### 음수 온도 측정 (확장 회로)

음수 온도 측정을 위해서는 GND 핀에 부(-) 전압을 공급하거나,
출력에 오프셋을 추가하는 회로가 필요합니다.

```
방법: 전압 분배기로 0.5V 오프셋 추가
  → Vout_shifted = Vout_LM35 + 500mV (오프셋)
  → 음수 측정 가능 (-50°C까지)
  → 소프트웨어에서 오프셋 역산
```

### 노이즈 감소 회로

```
ESP32-C3          LM35
  3.3V ─┬──────── VS  (1번)
        │
       100nF      (전원 바이패스 커패시터)
        │
  GND  ─┴──────── GND (3번)
  G1   ──┬──────── Vout(2번)
         │
        10nF      (ADC 입력 노이즈 필터)
         │
  GND  ──┘
```

### 코드 예시 (ESP32-C3)

```cpp
#define LM35_PIN   1    // ADC 핀 (GPIO1)
#define ADC_BITS   12   // ESP32-C3 ADC 12비트
#define ADC_VREF   3.3  // ADC 기준 전압 (V)
#define SAMPLES    64   // 평균 샘플 수 (노이즈 감소)

float readTemperature() {
  long sum = 0;
  for (int i = 0; i < SAMPLES; i++) {
    sum += analogRead(LM35_PIN);
    delayMicroseconds(100);
  }
  float avgRaw   = sum / (float)SAMPLES;
  float voltage  = avgRaw * ADC_VREF / ((1 << ADC_BITS) - 1); // V
  float tempC    = voltage * 100.0; // 10mV/°C → V×100 = °C
  return tempC;
}

void setup() {
  Serial.begin(115200);
  analogReadResolution(12);     // ESP32는 명시적 설정 권장
}

void loop() {
  float temp = readTemperature();
  Serial.printf("온도: %.2f °C\n", temp);
  delay(500);
}
```

## 측정값 계산 공식

### 기본 변환

```
Vout (V) = ADC_raw × VREF / (2^bits - 1)

온도 (°C) = Vout (mV) / 10
          = Vout (V) × 100
```

### ESP32-C3 (12비트 ADC, 3.3V 기준)

```
Vout = ADC_raw × 3.3 / 4095  (V)
T(°C) = Vout × 100
      = ADC_raw × 3.3 × 100 / 4095
      = ADC_raw × 0.08058... (°C/LSB)

분해능: 3.3V / 4095 / 0.010 = 0.0806°C (1 LSB)
```

### 오프셋 포함 (음수 온도)

```
오프셋 전압 Voffset = 0.5V (= 50°C 상당)
T(°C) = (Vout - Voffset) × 100
```

## 주의사항

1. **ESP32 ADC 비선형성**
   - ESP32 계열의 ADC는 비선형 특성이 있어 전압 범위 양 끝(0V 근처, VREF 근처)에서 오차가 큼
   - 0.1V ~ 3.1V 범위(약 1°C ~ 310°C 사이)에서 사용 권장
   - `esp_adc_cal` 라이브러리로 ADC 캘리브레이션 보정 가능

2. **자가 발열**
   - 밀폐 공간에서 1mA 이상 부하 전류가 흐르면 자가 발열로 오차 발생
   - 정적 공기 중 0.08°C/mW — 부하 최소화 필요

3. **긴 배선 노이즈**
   - 긴 배선은 EMI를 유도하여 ADC 오차를 유발
   - 배선 끝에 10~100nF 커패시터를 GND로 연결하여 고주파 노이즈 제거

4. **Wi-Fi와 ADC 간섭 (ESP32)**
   - ESP32 계열에서 Wi-Fi 활성화 시 ADC2 핀이 사용 불가
   - ADC1 핀(G0, G1, G2, G3, G4)을 사용해야 합니다 (ESP32-C3 기준)

5. **출력 단락 주의**
   - Vout 핀이 단락되면 IC 손상 가능
   - 출력에 직렬 저항(예: 470Ω)을 추가하면 단락 보호

6. **GND 바운싱**
   - ADC GND와 센서 GND를 동일 지점(스타 접지)에 연결하여 전위차 제거

## 캘리브레이션 방법

### ESP32 ADC 보정 (esp_adc_cal 사용)

```cpp
#include "esp_adc_cal.h"

esp_adc_cal_characteristics_t adcChars;
const adc_channel_t ADC_CHAN = ADC_CHANNEL_1; // GPIO1
const adc_atten_t   ADC_ATTEN = ADC_ATTEN_DB_11; // 0~3.9V 범위

void setup() {
  adc1_config_width(ADC_WIDTH_BIT_12);
  adc1_config_channel_atten((adc1_channel_t)ADC_CHAN, ADC_ATTEN);

  // 캘리브레이션 특성 로드 (eFuse 기반)
  esp_adc_cal_characterize(ADC_UNIT_1, ADC_ATTEN,
                           ADC_WIDTH_BIT_12, 1100,
                           &adcChars);
}

float readTempCalibrated() {
  int raw      = adc1_get_raw((adc1_channel_t)ADC_CHAN);
  uint32_t mV  = esp_adc_cal_raw_to_voltage(raw, &adcChars);
  return mV / 10.0; // mV / (10mV/°C) = °C
}
```

### 기준 온도계 비교 보정

```cpp
// 기준 온도계로 측정한 실제 온도를 입력하여 오프셋 계산
const float KNOWN_TEMP = 25.0;  // 기준 온도계 값
float measured = readTemperature();
float offset   = KNOWN_TEMP - measured;

// 이후 측정 시
float corrected = readTemperature() + offset;
```

## 실사용 팁

- **다중 샘플 평균**: 단일 ADC 읽기보다 16~64회 평균을 내면 노이즈가 크게 줄어듦
- **ADC 핀 선택**: ESP32-C3에서 Wi-Fi 사용 시 ADC1 핀만 사용 가능
- **DHT22와 비교**: LM35는 온도만 측정 가능하지만 응답 속도가 빠름 (수십 ms)
- **3.3V vs 5V 공급**: 3.3V 공급 시 최대 측정 온도 제한 없음 (LM35 자체 한계 150°C) — ADC 범위 내면 문제없음
- **직사광선 차피**: 센서를 알루미늄 포일 차광 케이스에 넣으면 복사열 영향 감소
- **간단한 회로**: 단 3개의 핀만 연결하면 되므로 교육용 및 프로토타입에 적합
