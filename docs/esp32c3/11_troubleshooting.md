# 트러블슈팅

## 업로드 문제

| 증상 | 원인 | 해결 |
|------|------|------|
| 포트가 안 보임 | 드라이버 미설치 | CH340 드라이버 설치 |
| 업로드 실패 (타임아웃) | 부팅 모드 오류 | G9(BOOT) 누른 채 리셋 후 업로드 |
| 업로드 느리거나 실패 | 속도 문제 | Upload Speed → `115200` 으로 낮추기 |

---

## 시리얼 모니터 문제

| 증상 | 원인 | 해결 |
|------|------|------|
| 아무것도 안 나옴 | USB CDC 비활성 | 보드 설정 `USB CDC On Boot: Enabled` |
| 깨진 문자 출력 | 통신속도 불일치 | `Serial.begin()` 속도와 모니터 속도 동일하게 |
| 리셋 후 첫 출력 잘림 | CDC 초기화 지연 | `delay(1000)` 을 `setup()` 맨 앞에 추가 |

## Wi-Fi 문제

| 증상 | 원인 | 해결 |
|------|------|------|
| 연결 안 됨 | 5GHz 공유기 | 2.4GHz 대역 사용 |
| 간헐적 끊김 | 전원 불안정 | 5V/500mA 이상 안정 전원 사용 |
| IP 못 받음 | DHCP 문제 | 고정 IP 설정으로 우회 |

```cpp
// 고정 IP 설정
IPAddress ip(192, 168, 1, 100);
IPAddress gateway(192, 168, 1, 1);
IPAddress subnet(255, 255, 255, 0);
WiFi.config(ip, gateway, subnet);
WiFi.begin(ssid, password);
```

## ADC 문제

| 증상 | 원인 | 해결 |
|------|------|------|
| 아날로그 값이 튐 | Wi-Fi 간섭 | Wi-Fi 끄거나 외부 ADC 사용 |
| 값이 0만 나옴 | 3.3V 초과 입력 | 분압 저항으로 전압 낮추기 |

---

## 전압 주의사항

```
⚠️ ESP32-C3는 3.3V 로직입니다.
   GPIO에 5V 직접 연결 시 보드 손상!

   5V 센서 사용 시:
   → 레벨 시프터 사용
   → 또는 분압 저항 (10kΩ + 20kΩ)
```

## 기타

| 증상 | 해결 |
|------|------|
| 루프가 계속 재시작됨 (Watchdog) | `loop()` 안에 긴 `delay()` 제거, `yield()` 추가 |
| 메모리 부족 오류 | `String` 대신 `char[]` 사용, 전역변수 줄이기 |
| 플래시 쓰기 횟수 한계 | NVS 쓰기는 변경될 때만 수행 (매 루프 쓰기 금지) |
