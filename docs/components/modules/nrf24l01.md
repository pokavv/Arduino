# nRF24L01+ 2.4GHz 무선 통신 모듈

## 개요

nRF24L01+는 Nordic Semiconductor의 2.4GHz ISM 대역 트랜시버 IC를 사용한 무선 통신 모듈입니다.
블루투스와 달리 페어링 과정 없이 채널과 주소만 맞추면 즉시 통신이 가능하고,
한 마스터(PTX)가 최대 6개 슬레이브(PRX)와 동시에 통신하는 멀티포인트 네트워크를 지원합니다.

특징 요약:
- SPI 인터페이스로 Arduino와 연결 (속도 빠름)
- 최대 250kbps ~ 2Mbps 전송 속도
- 낮은 소비 전력 (수신 대기 시 약 12mA)
- 자동 재전송(Auto-Retransmit), 자동 ACK, 6채널 멀티포인트
- 저렴한 가격 (모듈 기준 수백 원)

Arduino 프로젝트에서의 활용:
- RC카/드론 원격 제어 (조이스틱 → RC카)
- 다중 센서 네트워크 (6개 센서 노드 → 1개 게이트웨이)
- 무선 데이터 수집 (온도, 습도 등)
- Arduino 간 단거리 무선 통신

## 핀 구성

### 8핀 DIP 모듈 (표준)

```
핀 배열 (2×4 핀 헤더):
  [GND] [VCC]
  [CE ] [CSN]
  [SCK] [MOSI]
  [MISO][IRQ ]
```

| 핀 | 이름 | 방향 | 기능 |
|----|------|------|------|
| 1 | GND | 전원 | 공통 접지 |
| 2 | VCC | 전원 | 전원 입력 (1.9~3.6V, 반드시 3.3V!) |
| 3 | CE | 입력 | Chip Enable. TX/RX 모드 전환 |
| 4 | CSN | 입력 | SPI Chip Select (Low = 활성) |
| 5 | SCK | 입력 | SPI 클럭 |
| 6 | MOSI | 입력 | SPI Master Out Slave In (Arduino → nRF) |
| 7 | MISO | 출력 | SPI Master In Slave Out (nRF → Arduino) |
| 8 | IRQ | 출력 | 인터럽트 출력 (RX 수신 완료, TX 완료, 최대 재전송 시 LOW) |

## 동작 원리

### 주요 동작 모드

| 모드 | CE | CSN | 설명 |
|------|----|-----|------|
| Power Down | X | X | 최저 전력, 900nA |
| Standby-I | LOW | X | SPI 가능, TX FIFO 비어있음, 22μA |
| Standby-II | HIGH | X | TX FIFO에 데이터 있음, 대기 |
| RX Mode | HIGH | — | 수신 대기, 12mA |
| TX Mode | HIGH(펄스) | — | 송신 중, 11mA |

### ShockBurst 프로토콜

nRF24L01+의 핵심 기능:
1. **자동 패킷 조립**: 프리앰블, 주소, 페이로드, CRC를 자동 추가
2. **자동 ACK**: 수신 측이 자동으로 ACK 전송, 송신 측이 확인
3. **자동 재전송**: ACK 미수신 시 설정된 횟수만큼 자동 재전송
4. **CRC 오류 감지**: 손상된 패킷 자동 폐기

## 아두이노 연결 방법

### 5V Arduino와 연결 (레벨 변환 필수!)

```
nRF24L01 VCC  ──── 3.3V (Arduino 3.3V 핀 또는 AMS1117-3.3V)
nRF24L01 GND  ──── GND
nRF24L01 CE   ──── Arduino 핀 9
nRF24L01 CSN  ──── Arduino 핀 10
nRF24L01 SCK  ──── Arduino 핀 13 (SPI SCK)
nRF24L01 MOSI ──── Arduino 핀 11 (SPI MOSI)
nRF24L01 MISO ──── Arduino 핀 12 (SPI MISO)
nRF24L01 IRQ  ──── Arduino 핀 2 (인터럽트, 옵션)
```

> 5V Arduino에서 SPI 신호(SCK, MOSI, CSN, CE)는 5V 출력이므로 3.3V 내성인 nRF24L01+에 위험할 수 있습니다.
> 실제로 많은 모듈에 5V 내성이 있다고 알려져 있지만, 안전하게 74HC125 또는 레벨 변환 모듈 사용을 권장합니다.

### ESP32-C3 와 연결

```
nRF24L01 VCC  ──── ESP32-C3 3V3
nRF24L01 GND  ──── GND
nRF24L01 CE   ──── GPIO4
nRF24L01 CSN  ──── GPIO5
nRF24L01 SCK  ──── GPIO6  (SPI SCK)
nRF24L01 MOSI ──── GPIO7  (SPI MOSI)
nRF24L01 MISO ──── GPIO2  (SPI MISO)
              (모두 3.3V 레벨, 변환 불필요)
```

### 전원 안정화 (캐패시터 필수!)

```
nRF24L01 VCC ─┬─── 3.3V
               │
            [10~100μF 전해] + [100nF 세라믹] (병렬)
               │
              GND
```

> nRF24L01+는 송신 시 순간적으로 전류가 급증합니다. 캐패시터 없으면 전압이 순간 떨어져 통신 불안정 또는 Arduino 리셋 발생. 반드시 캐패시터 추가!

## 설정 방법 (RF24 라이브러리)

### 라이브러리 설치

```
Arduino IDE → 라이브러리 관리자 → "RF24" 검색
→ TMRh20의 "RF24" 설치 (가장 널리 사용)
```

### 송신측 (PTX) 코드

```cpp
#include <SPI.h>
#include <RF24.h>

// CE=9, CSN=10
RF24 radio(9, 10);

// 5바이트 파이프 주소 (송신/수신 측이 동일해야 함)
const byte PIPE_ADDRESS[6] = "00001";

void setup() {
    Serial.begin(115200);
    radio.begin();

    // 송신 출력 설정 (RF24_PA_MIN, RF24_PA_LOW, RF24_PA_HIGH, RF24_PA_MAX)
    radio.setPALevel(RF24_PA_LOW);

    // 전송 속도 (RF24_250KBPS, RF24_1MBPS, RF24_2MBPS)
    radio.setDataRate(RF24_1MBPS);

    // 채널 설정 (0~125, 2400~2525 MHz)
    radio.setChannel(76);

    // 자동 ACK 활성화 (기본값 ON)
    radio.setAutoAck(true);

    // 재전송 설정 (지연 5×250μs=1250μs, 최대 15회)
    radio.setRetries(5, 15);

    // 송신 파이프 열기
    radio.openWritingPipe(PIPE_ADDRESS);
    radio.stopListening();  // 송신 모드
    Serial.println("송신 준비 완료");
}

void loop() {
    int sensorVal = analogRead(A0);
    bool ok = radio.write(&sensorVal, sizeof(sensorVal));

    if (ok) {
        Serial.print("전송 성공: ");
        Serial.println(sensorVal);
    } else {
        Serial.println("전송 실패 (ACK 없음)");
    }
    delay(500);
}
```

### 수신측 (PRX) 코드

```cpp
#include <SPI.h>
#include <RF24.h>

RF24 radio(9, 10);  // CE=9, CSN=10
const byte PIPE_ADDRESS[6] = "00001";

void setup() {
    Serial.begin(115200);
    radio.begin();
    radio.setPALevel(RF24_PA_LOW);
    radio.setDataRate(RF24_1MBPS);
    radio.setChannel(76);
    radio.setAutoAck(true);

    // 수신 파이프 0번에서 열기
    radio.openReadingPipe(0, PIPE_ADDRESS);
    radio.startListening();  // 수신 모드
    Serial.println("수신 대기 중...");
}

void loop() {
    if (radio.available()) {
        int received;
        radio.read(&received, sizeof(received));
        Serial.print("수신 데이터: ");
        Serial.println(received);
    }
}
```

## 주의사항

1. **전원 3.3V 절대 준수**: nRF24L01+의 VCC 최대 허용 전압은 3.6V. 5V를 연결하면 즉시 파손됩니다.
2. **전원 캐패시터 필수**: 10μF 이상 캐패시터를 VCC-GND 사이에 반드시 추가. 미설치 시 통신 오류의 80%를 차지.
3. **Wi-Fi 간섭**: Wi-Fi(2.4GHz), 블루투스와 같은 대역을 사용합니다. 채널을 Wi-Fi와 겹치지 않는 범위로 설정 (76~83번 채널 권장).
4. **SPI 속도**: 기본 SPI 속도(4MHz~8MHz)로 사용. 너무 빠른 SPI 속도는 통신 오류 유발.
5. **IRQ 핀 폴링 vs 인터럽트**: 단순 구현 시 radio.available() 폴링으로 충분. 저전력 최적화 시 IRQ 인터럽트 사용.
6. **페이로드 크기**: 기본 32바이트 한계. 큰 데이터는 여러 패킷으로 나눠 전송.

## 실사용 팁 및 회로 패턴

### 통신 거리 최적화

| 설정 | 속도 | 거리 | 전력 |
|------|------|------|------|
| RF24_250KBPS + RF24_PA_MAX | 250kbps | 최대 | 최고 |
| RF24_1MBPS + RF24_PA_LOW | 1Mbps | 중간 | 낮음 |
| RF24_2MBPS + RF24_PA_MIN | 2Mbps | 최소 | 최소 |

### 6채널 멀티포인트 수신

```cpp
// 하나의 수신기가 6개 송신기를 동시에 수신
const byte PIPES[6][6] = {
    "SENS1", "SENS2", "SENS3",
    "SENS4", "SENS5", "SENS6"
};

for (int i = 0; i < 6; i++) {
    radio.openReadingPipe(i, PIPES[i]);
}
radio.startListening();

// 수신 시 어떤 파이프에서 왔는지 확인
uint8_t pipeNum;
if (radio.available(&pipeNum)) {
    int data;
    radio.read(&data, sizeof(data));
    Serial.print("파이프 ");
    Serial.print(pipeNum);
    Serial.print("에서 수신: ");
    Serial.println(data);
}
```

### 저전력 수면 구현

```cpp
// 데이터 전송 후 Power Down 모드로 전환
radio.powerDown();
// ESP32 딥슬립
esp_deep_sleep(30 * 1000000);  // 30초 후 깨어남

// 깨어난 후
radio.powerUp();
delay(5);  // 5ms 안정화 대기
radio.stopListening();
// 데이터 전송
```
