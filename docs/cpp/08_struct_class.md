# 구조체와 클래스

## 구조체 (struct)

### 구조체가 뭔가요?

서로 관련된 변수들을 하나로 묶는 사용자 정의 자료형입니다.

```cpp
// 온도/습도/기압을 항상 같이 다룬다면?
// 변수 3개를 따로 관리하는 대신
float 온도, 습도, 기압;

// 구조체로 묶으면 하나처럼 다룰 수 있음
struct 날씨데이터 {
    float 온도;
    float 습도;
    float 기압;
};
```

### 구조체 선언과 사용

```cpp
struct 센서데이터 {
    float 온도;
    float 습도;
    int   밝기;
    bool  유효함;
};

// 변수 생성
센서데이터 측정값;

// 멤버 접근: 변수이름.멤버이름
측정값.온도  = 25.3;
측정값.습도  = 60.0;
측정값.밝기  = 512;
측정값.유효함 = true;

Serial.println(측정값.온도);
```

### 초기화

```cpp
// 선언과 동시에 초기화
센서데이터 초기값 = {25.0, 50.0, 300, false};

// 또는 이름으로 초기화 (C++11 이상)
센서데이터 초기값 = {
    .온도   = 25.0,
    .습도   = 50.0,
    .밝기   = 300,
    .유효함  = false
};
```

### 함수에 구조체 넘기기

```cpp
void 데이터출력(센서데이터 데이터) {
    Serial.print("온도: "); Serial.println(데이터.온도);
    Serial.print("습도: "); Serial.println(데이터.습도);
}

센서데이터 출력(측정값);  // 구조체를 통째로 전달
```

### 구조체 포인터 (→ 연산자)

포인터로 구조체에 접근할 때는 `.` 대신 `->` 를 씁니다.

```cpp
센서데이터* p = &측정값;
p->온도 = 30.0;    // (*p).온도 = 30.0 과 동일
```

## 클래스 (class)

### 클래스가 뭔가요?

구조체에서 한 단계 더 나아가, **데이터(변수)와 동작(함수)을 함께 묶은 것**입니다.
아두이노 라이브러리 대부분이 클래스로 만들어져 있습니다.

```cpp
// Servo servo;          → servo는 Servo 클래스의 객체
// servo.attach(9);      → 클래스의 함수(메서드) 호출
// servo.write(90);      → 클래스의 함수(메서드) 호출
```

### 클래스 선언

```cpp
class LED제어기 {
private:                    // 외부에서 접근 불가
    int 핀번호;
    bool 켜짐상태;

public:                     // 외부에서 접근 가능
    // 생성자: 객체가 만들어질 때 자동 실행
    LED제어기(int 핀) {
        핀번호 = 핀;
        켜짐상태 = false;
        pinMode(핀번호, OUTPUT);
        digitalWrite(핀번호, HIGH);
    }

    void 켜기() {
        digitalWrite(핀번호, LOW);
        켜짐상태 = true;
    }

    void 끄기() {
        digitalWrite(핀번호, HIGH);
        켜짐상태 = false;
    }

    void 토글() {
        if (켜짐상태) 끄기();
        else 켜기();
    }

    bool 상태확인() {
        return 켜짐상태;
    }
};
```

### 클래스 사용 (객체 생성)

```cpp
// 객체 생성 (생성자 자동 호출)
LED제어기 내장LED(8);         // 8번 핀 LED
LED제어기 외부LED(4);         // 4번 핀 LED

void setup() { }   // 생성자에서 이미 처리함

void loop() {
    내장LED.켜기();
    delay(500);
    내장LED.끄기();
    delay(500);

    외부LED.토글();    // 상태 반전
    delay(250);
}
```

## 구조체 vs 클래스

| 항목 | struct | class |
|------|--------|-------|
| 기본 접근 | public (외부 접근 가능) | private (외부 접근 불가) |
| 함수 포함 | 가능하지만 잘 안 씀 | 주로 함수와 함께 씀 |
| 주로 쓰는 경우 | 데이터 묶음 | 동작 포함한 객체 |

## 아두이노 라이브러리가 클래스인 이유

```cpp
// 클래스 덕분에 같은 종류의 장치를 여러 개 독립적으로 다룰 수 있음

Servo 서보1, 서보2, 서보3;   // 세 개의 독립적인 서보모터

서보1.attach(9);
서보2.attach(10);
서보3.attach(11);

서보1.write(0);    // 서보1만 0도
서보2.write(90);   // 서보2만 90도
서보3.write(180);  // 서보3만 180도
```

각 객체는 자기 자신의 상태(핀 번호, 각도 등)를 따로 관리합니다.
