# 74HC595 직렬-병렬 시프트 레지스터

## 개요

74HC595는 직렬 입력(SPI)을 8비트 병렬 출력으로 변환하는 CMOS 시프트 레지스터 IC입니다. 아두이노의 디지털 핀 3개만으로 8개의 출력 핀을 제어할 수 있으며, 여러 개를 직렬 연결(데이지 체인)하면 3개 핀으로 16개, 24개, 32개 등 무제한 출력 확장이 가능합니다.

**주요 용도**: LED 매트릭스, 7-세그먼트 디스플레이 멀티플렉싱, 다채널 릴레이 제어, 핀 확장.

**핵심 특징**:
- 3선(SPI) 인터페이스로 8비트 출력 제어
- 데이지 체인(직렬 연결)으로 무한 확장
- 출력 래치(OE 핀)로 깜박임 없는 갱신 가능

## 핀/단자 구성

### 74HC595 DIP-16 패키지

| 핀 번호 | 핀 이름 | 기능 |
|--------|---------|------|
| 1 | QB (Q1) | 직렬 출력 비트 1 |
| 2 | QC (Q2) | 직렬 출력 비트 2 |
| 3 | QD (Q3) | 직렬 출력 비트 3 |
| 4 | QE (Q4) | 직렬 출력 비트 4 |
| 5 | QF (Q5) | 직렬 출력 비트 5 |
| 6 | QG (Q6) | 직렬 출력 비트 6 |
| 7 | QH (Q7) | 직렬 출력 비트 7 (MSB 또는 LSB에 따라) |
| 8 | GND | 접지 |
| 9 | QH' | 직렬 출력 (다음 595로 데이지 체인) |
| 10 | SRCLR (MR) | 시프트 레지스터 클리어 (Active LOW) |
| 11 | SRCLK (SH_CP) | 시프트 레지스터 클럭 (상승 에지에 데이터 이동) |
| 12 | RCLK (ST_CP) | 래치 클럭/스토리지 클럭 (상승 에지에 출력 갱신) |
| 13 | OE (G) | 출력 활성화 (Active LOW, LOW=출력, HIGH=하이 임피던스) |
| 14 | SER (DS) | 직렬 데이터 입력 |
| 15 | QA (Q0) | 병렬 출력 비트 0 (LSB) |
| 16 | VCC | 전원 (+3.3V 또는 +5V) |

## 동작 원리

74HC595 내부는 두 단계 레지스터로 구성됩니다.

```
SER(14) ──→ [8비트 시프트 레지스터] ──→ SRCLK(11)으로 비트 이동
                    ↓ RCLK(12) 상승 에지
              [8비트 스토리지 레지스터] ──→ QA~QH (출력)
                                                ↑ OE(13) = LOW 시 출력 활성
```

**데이터 전송 과정:**
1. **시프트 클럭(SRCLK)**: 상승 에지마다 SER 핀의 현재 값을 시프트 레지스터로 이동. 기존 데이터는 한 칸씩 밀림.
2. **8번 반복**: 비트 0~7을 순서대로 SER에 설정 후 SRCLK 펄스 → 8비트 모두 로드.
3. **래치 클럭(RCLK)**: 상승 에지에 시프트 레지스터 내용을 스토리지 레지스터로 복사 → QA~QH 출력 갱신.
4. **OE**: LOW이면 스토리지 레지스터 내용이 출력 핀에 나타남. HIGH이면 출력 하이 임피던스(3-state).

**데이지 체인**: QH' 핀은 시프트 레지스터의 마지막 비트가 나오는 직렬 출력. 이 핀을 다음 595의 SER에 연결하면 SRCLK/RCLK을 공유하며 16비트, 24비트로 확장 가능.

## 진리표 (Truth Table)

### 시프트 레지스터 동작

| SRCLR | SRCLK | SER | 동작 |
|-------|-------|-----|------|
| LOW | X | X | 시프트 레지스터 전체 클리어 (0으로) |
| HIGH | ↑ | HIGH | '1'을 Q0에 입력, 기존 데이터 한 칸 상위로 시프트 |
| HIGH | ↑ | LOW | '0'을 Q0에 입력, 기존 데이터 한 칸 상위로 시프트 |
| HIGH | LOW/HIGH | X | 시프트 없음 (클럭 엣지 없으므로) |

### 래치 및 출력 동작

| RCLK | OE | 동작 |
|------|-----|------|
| ↑ | LOW | 시프트 레지스터 → 스토리지 복사 → 출력 갱신 |
| LOW/HIGH | LOW | 출력 유지 |
| X | HIGH | 출력 핀 하이 임피던스 (3-state) |

## 아두이노 연결 방법

### 기본 연결 (1개)

```
ESP32-C3 Super Mini       74HC595
───────────────────       ────────
3.3V ─────────────────→ VCC (16)
GND  ─────────────────→ GND (8)
GND  ─────────────────→ OE (13)  ← 항상 LOW = 출력 활성
3.3V ─────────────────→ SRCLR (10) ← 항상 HIGH = 클리어 안 함
G4 (DATA) ────────────→ SER (14)
G5 (CLK)  ────────────→ SRCLK (11)
G6 (LATCH)────────────→ RCLK (12)

QA(15) ~ QH(7) → LED 또는 부하 (각 핀에 220Ω 저항)
```

```

────────           ──────────           ──────────

G5(CLK)  ─────────────→ SRCLK(11) ──→ SRCLK(11)
G6(LATCH)─────────────→ RCLK(12)  ──→ RCLK(12)
VCC ──────────────────→ VCC        ──→ VCC
GND ──────────────────→ GND        ──→ GND
```

## 제어 방법 (아두이노 코드)

### shiftOut() 함수 사용

```cpp
const int DATA_PIN  = 4;   // SER
const int CLK_PIN   = 5;   // SRCLK
const int LATCH_PIN = 6;   // RCLK

void setup() {
  pinMode(DATA_PIN,  OUTPUT);
  pinMode(CLK_PIN,   OUTPUT);
  pinMode(LATCH_PIN, OUTPUT);
  digitalWrite(LATCH_PIN, LOW);
}

void writeShiftReg(byte data) {
  digitalWrite(LATCH_PIN, LOW);           // 래치 잠금 (갱신 일시 중단)
  shiftOut(DATA_PIN, CLK_PIN, MSBFIRST, data);  // MSB부터 전송
  digitalWrite(LATCH_PIN, HIGH);          // 래치 열기 (출력 갱신)
  digitalWrite(LATCH_PIN, LOW);
}

void loop() {
  // LED 순차 점등
  for (int i = 0; i < 8; i++) {
    writeShiftReg(1 << i);  // 비트 하나씩 ON
    delay(200);
  }
  writeShiftReg(0xFF);  // 전체 ON
  delay(500);
  writeShiftReg(0x00);  // 전체 OFF
  delay(500);
}
```

### 2개 데이지 체인 (16비트)

```cpp
void write16(uint16_t data) {
  digitalWrite(LATCH_PIN, LOW);
  shiftOut(DATA_PIN, CLK_PIN, MSBFIRST, (byte)(data >> 8));  // 상위 8비트 (#2)
  shiftOut(DATA_PIN, CLK_PIN, MSBFIRST, (byte)(data & 0xFF)); // 하위 8비트 (#1)
  digitalWrite(LATCH_PIN, HIGH);
  digitalWrite(LATCH_PIN, LOW);
}
```

### SPI 라이브러리를 이용한 고속 전송

```cpp
#include <SPI.h>

const int LATCH_PIN = 6;
// ESP32 기본 SPI: MOSI=G23(또는 보드별 상이), SCK=G18

void writeShiftRegSPI(byte data) {
  SPI.beginTransaction(SPISettings(8000000, MSBFIRST, SPI_MODE0));
  digitalWrite(LATCH_PIN, LOW);
  SPI.transfer(data);
  digitalWrite(LATCH_PIN, HIGH);
  SPI.endTransaction();
  digitalWrite(LATCH_PIN, LOW);
}
```

### 7-세그먼트 디스플레이 출력

```cpp
// 7-세그먼트 공통 음극(CC) 기준 세그먼트 맵
// 세그먼트: a, b, c, d, e, f, g, dp
//           QA, QB, QC, QD, QE, QF, QG, QH
const byte DIGIT_MAP[] = {
  0b00111111,  // 0
  0b00000110,  // 1
  0b01011011,  // 2
  0b01001111,  // 3
  0b01100110,  // 4
  0b01101101,  // 5
  0b01111101,  // 6
  0b00000111,  // 7
  0b01111111,  // 8
  0b01101111,  // 9
};

void showDigit(int n) {
  writeShiftReg(DIGIT_MAP[n % 10]);
}
```

## 주의사항

1. **전류 제한**: 핀당 최대 35mA, 전체 70mA. LED 직결 시 반드시 220~470Ω 직렬 저항 사용.
2. **VCC 디커플링**: VCC와 GND 사이에 0.1µF 세라믹 커패시터를 IC에 최대한 가깝게 배치. 고속 스위칭 노이즈 억제.
3. **래치 타이밍**: `shiftOut()` 전에 RCLK(LATCH)를 LOW로, 전송 완료 후 HIGH 펄스. 중간에 래치 펄스 없을 것.
4. **SRCLR 핀**: 반드시 VCC에 연결 (또는 HIGH 유지). 실수로 LOW가 되면 레지스터 초기화.
5. **OE 핀**: GND에 연결 고정 (항상 출력 활성). 깜박임 없이 전환하려면 OE를 HIGH→데이터 로드→LOW 순서.
6. **데이지 체인 순서**: `shiftOut()` 첫 번째 바이트가 가장 멀리 있는 IC의 출력이 됨 (마지막으로 밀려나므로).
7. **3.3V 호환**: 74HC595는 VCC=3.3V에서 동작 가능. ESP32-C3 직결 가능.

## 자주 쓰이는 회로 패턴

### 패턴 1: LED 바 그래프 (볼륨 미터)

```cpp
void showLevel(int level) {  // level: 0~8
  byte mask = 0;
  for (int i = 0; i < level; i++) mask |= (1 << i);
  writeShiftReg(mask);
}
// ADC → 0~8 변환 → showLevel()
```

### 패턴 2: 4자리 7-세그 멀티플렉싱 (595 + 트랜지스터)

```cpp
// 595 → 세그먼트(a~g), 다른 GPIO 4개 → 자릿수 선택 트랜지스터
void displayNumber(int num) {
  int digits[4] = {num/1000, (num/100)%10, (num/10)%10, num%10};
  for (int d = 0; d < 4; d++) {
    selectDigit(d);
    writeShiftReg(DIGIT_MAP[digits[d]]);
    delay(5);
    deselectAll();
  }
}
```

### 패턴 3: 8채널 릴레이 제어 (ULN2003과 조합)

```
595 QA~QH → ULN2003 IN1~IN7 → 릴레이 코일 8개
단 3개 GPIO로 8채널 릴레이 제어 가능
```

### 패턴 4: OE 핀으로 부드러운 전환

```cpp
void writeNoFlicker(byte data) {
  digitalWrite(OE_PIN, HIGH);    // 출력 끄기
  shiftOut(DATA_PIN, CLK_PIN, MSBFIRST, data);  // 데이터 로드
  digitalWrite(LATCH_PIN, HIGH); // 래치
  digitalWrite(LATCH_PIN, LOW);
  digitalWrite(OE_PIN, LOW);     // 출력 켜기
}
```
