# TM1637 7-세그먼트 디스플레이 드라이버

## 개요

TM1637은 Titan Micro Electronics가 개발한 LED 세그먼트 디스플레이 전용 드라이버 IC입니다. 자체 통신 프로토콜(I2C와 유사하지만 별개)을 사용하며, CLK/DIO 2개 핀만으로 최대 6자리 7-세그먼트 디스플레이와 키패드 스캔(6열 × 8행)을 동시에 제어할 수 있습니다. 아두이노 4자리 클럭 디스플레이 모듈에 가장 많이 사용됩니다.

**주요 특징:**
- 2-wire 통신 (CLK + DIO)
- 최대 6자리 디스플레이 구동
- 밝기 8단계 조절 가능
- LED 드라이버 전류 내장 (외부 저항 불필요)
- 내장 키스캔 인터페이스

## 핀/단자 구성

### TM1637 IC (SOP-20 패키지)

| 핀 번호 | 핀 이름 | 기능 |
|--------|---------|------|
| 1 | VDD | 전원 (3.3 ~ 5.5 V) |
| 2 | K1 | 키 스캔 입력 1 |
| 3 | K2 | 키 스캔 입력 2 |
| 4 | GND | 접지 |
| 5~14 | SEG1~SEG8 | 세그먼트 출력 (a,b,c,d,e,f,g,dp) |
| 15~20 | GRID1~GRID6 | 자릿수 선택 출력 (공통 단자 구동) |
| — | CLK | 통신 클럭 입력 |
| — | DIO | 통신 데이터 입력/출력 |

### 4자리 TM1637 모듈 단자

| 단자 | 기능 |
|------|------|
| CLK | 클럭 신호 (아두이노 디지털 핀) |
| DIO | 데이터 신호 (아두이노 디지털 핀) |
| VCC | 전원 (3.3V 또는 5V) |
| GND | 접지 |

## 동작 원리

TM1637은 멀티플렉싱(시분할) 방식으로 6자리를 구동합니다.

**통신 프로토콜 (I2C 유사, 호환 아님):**

```
시작 조건: CLK=HIGH 상태에서 DIO를 HIGH→LOW
데이터:    CLK LOW 구간에 DIO 설정, CLK ↑ 상승 에지에 샘플링 (LSB 먼저)
ACK:       8비트 전송 후 CLK ↓ 시 DIO 입력 전환, CLK ↑ 시 ACK 샘플링
종료 조건: CLK=LOW에서 DIO=LOW 설정 → CLK HIGH → DIO HIGH
```

**명령어 구조:**
1. **데이터 명령 (0x40)**: 쓰기 모드, 자동/고정 주소 선택
2. **주소 명령 (0xC0~0xC5)**: 6개 자릿수(레지스터) 주소
3. **디스플레이 제어 명령 (0x88~0x8F)**: 밝기 및 ON/OFF

**멀티플렉싱:**  
내부 타이머가 GRID1~GRID6을 순차적으로 활성화하며, 각 자릿수의 세그먼트 데이터를 SEG 핀으로 출력. 빠른 전환으로 모든 자릿수가 동시에 켜진 것처럼 보임.

## 진리표 - 세그먼트 코드

```
    _
   |_|
   |_|

세그먼트 배치:
     a
   ──────
f |      | b
   ──────   ← g
e |      | c
   ──────
     d      (dp: 소수점)
```

| 비트 | 세그먼트 |
|------|---------|
| bit0 (0x01) | a (상단 가로) |
| bit1 (0x02) | b (우상 세로) |
| bit2 (0x04) | c (우하 세로) |
| bit3 (0x08) | d (하단 가로) |
| bit4 (0x10) | e (좌하 세로) |
| bit5 (0x20) | f (좌상 세로) |
| bit6 (0x40) | g (중간 가로) |
| bit7 (0x80) | dp (소수점) |

## 아두이노 연결 방법

```
ESP32-C3 Super Mini       TM1637 모듈
───────────────────       ───────────
3.3V 또는 5V ─────────→ VCC
GND          ─────────→ GND
G4           ─────────→ CLK
G5           ─────────→ DIO
```

> TM1637 모듈은 3.3V와 5V 모두 동작. ESP32-C3의 3.3V 직결 가능.  
> CLK/DIO 풀업 저항(10kΩ)이 모듈에 내장된 경우 많음. 없으면 추가.

## 제어 방법 (아두이노 코드)

### TM1637Display 라이브러리 사용

```cpp
#include <TM1637Display.h>

const int CLK_PIN = 4;
const int DIO_PIN = 5;

TM1637Display display(CLK_PIN, DIO_PIN);

void setup() {
  display.setBrightness(0x0F);  // 최대 밝기 (0x00~0x0F)
}

void loop() {
  // 정수 표시
  display.showNumberDec(1234, false);  // 1234, 앞자리 0 없음
  delay(1000);

  // 앞자리 0 포함
  display.showNumberDec(42, true);     // 0042
  delay(1000);

  // 소수점/콜론 (시계 형식)
  display.showNumberDecEx(1230, 0b01000000, false);  // 12:30 (콜론)
  delay(1000);

  // 개별 세그먼트 제어
  uint8_t data[] = {
    SEG_A | SEG_B | SEG_C | SEG_D | SEG_E | SEG_F,  // 0
    SEG_B | SEG_C,                                     // 1
    0x40,                                              // - (마이너스)
    0x00                                               // (빈칸)
  };
  display.setSegments(data);
  delay(1000);

  // 특정 자릿수만 갱신
  display.setSegments(data, 2, 1);  // 2글자, 1번 자리부터
  delay(1000);
}
```

### 직접 통신 구현 (라이브러리 없이)

```cpp
const int CLK = 4;
const int DIO = 5;

const byte DIGIT_MAP[] = {
  0x3F, 0x06, 0x5B, 0x4F, 0x66,  // 0~4
  0x6D, 0x7D, 0x07, 0x7F, 0x6F   // 5~9
};

void start() {
  pinMode(DIO, OUTPUT);
  digitalWrite(DIO, HIGH);
  digitalWrite(CLK, HIGH);
  delayMicroseconds(2);
  digitalWrite(DIO, LOW);
  delayMicroseconds(2);
  digitalWrite(CLK, LOW);
}

void stop() {
  pinMode(DIO, OUTPUT);
  digitalWrite(CLK, LOW);
  digitalWrite(DIO, LOW);
  delayMicroseconds(2);
  digitalWrite(CLK, HIGH);
  delayMicroseconds(2);
  digitalWrite(DIO, HIGH);
}

void writeByte(byte b) {
  for (int i = 0; i < 8; i++) {
    digitalWrite(CLK, LOW);
    digitalWrite(DIO, (b >> i) & 0x01);  // LSB first
    delayMicroseconds(2);
    digitalWrite(CLK, HIGH);
    delayMicroseconds(2);
  }
  // ACK
  digitalWrite(CLK, LOW);
  pinMode(DIO, INPUT);
  delayMicroseconds(5);
  digitalWrite(CLK, HIGH);
  delayMicroseconds(2);
  digitalWrite(CLK, LOW);
  pinMode(DIO, OUTPUT);
}

void showNumber(int num) {
  byte d[4];
  d[3] = DIGIT_MAP[num / 1000 % 10];
  d[2] = DIGIT_MAP[num / 100  % 10];
  d[1] = DIGIT_MAP[num / 10   % 10];
  d[0] = DIGIT_MAP[num        % 10];

  // 데이터 명령: 자동 주소 증가, 쓰기 모드
  start();
  writeByte(0x40);
  stop();

  // 주소 명령 + 데이터 4바이트
  start();
  writeByte(0xC0);  // 시작 주소 0
  for (int i = 0; i < 4; i++) writeByte(d[i]);
  stop();

  // 디스플레이 제어: ON, 밝기 최대
  start();
  writeByte(0x8F);
  stop();
}
```

### 시계 표시 (콜론 깜박임)

```cpp
#include <TM1637Display.h>
TM1637Display display(4, 5);

void showClock(int hour, int minute, bool colonOn) {
  int time = hour * 100 + minute;
  display.showNumberDecEx(time, colonOn ? 0b01000000 : 0, true);
}

void loop() {
  static unsigned long lastToggle = 0;
  static bool colon = true;

  if (millis() - lastToggle >= 500) {
    lastToggle = millis();
    colon = !colon;
  }
  showClock(12, 30, colon);
}
```

## 주의사항

1. **전용 프로토콜**: TM1637은 표준 I2C가 아님. Wire 라이브러리 사용 불가. 전용 라이브러리(TM1637Display) 사용.
2. **LSB 먼저**: 표준 I2C(MSB 먼저)와 반대로 LSB 먼저 전송. 직접 구현 시 반드시 주의.
3. **CLK/DIO 풀업**: 통신 중 DIO는 양방향. 모듈에 내장 풀업(10kΩ) 없으면 외부 추가.
4. **밝기와 전류**: 밝기를 높이면 전류 소비 증가. 최대 밝기에서 5V 4자리 디스플레이는 100~200mA 소비.
5. **콜론 위치**: 4자리 모듈에서 콜론은 2번째 자릿수(인덱스 1)의 dp 비트로 제어. 모듈마다 위치 다를 수 있음.
6. **클럭 속도**: TM1637은 최대 500kHz이나, 소프트웨어 I2C 방식이므로 너무 빠르면 오동작. 기본 라이브러리 속도(약 250kHz) 사용 권장.
7. **음수 표시**: 음수는 맨 앞 자리에 `-` (0x40) 세그먼트 코드를 수동으로 배치.

## 자주 쓰이는 회로 패턴

### 패턴 1: 카운트다운 타이머

```cpp
void countDown(int seconds) {
  for (int s = seconds; s >= 0; s--) {
    int min = s / 60;
    int sec = s % 60;
    showClock(min, sec, true);
    delay(1000);
  }
  display.clear();
}
```

### 패턴 2: 온도 표시 (소수점 포함)

```cpp
void showTemp(float temp) {
  int t = (int)(temp * 10);  // 소수 첫째 자리까지
  // 예: 23.5°C → 235
  uint8_t segs[4];
  segs[3] = display.encodeDigit(t / 100);
  segs[2] = display.encodeDigit((t / 10) % 10) | 0x80;  // 소수점
  segs[1] = display.encodeDigit(t % 10);
  segs[0] = 0x39;  // C
  display.setSegments(segs);
}
```

### 패턴 3: 자릿수별 개별 업데이트

```cpp
// 우측 2자리만 갱신 (분 표시)
void updateMinutes(int min) {
  uint8_t segs[2];
  segs[0] = display.encodeDigit(min / 10);
  segs[1] = display.encodeDigit(min % 10);
  display.setSegments(segs, 2, 2);  // 2글자, 2번 자리부터
}
```

### 패턴 4: 스크롤 텍스트

```cpp
const uint8_t HELLO[] = {
  SEG_B|SEG_C|SEG_E|SEG_F|SEG_G,           // H
  SEG_A|SEG_D|SEG_E|SEG_F|SEG_G,           // E
  SEG_D|SEG_E|SEG_F,                        // L
  SEG_D|SEG_E|SEG_F,                        // L
  SEG_A|SEG_B|SEG_C|SEG_D|SEG_E|SEG_F,     // O
};

void scrollText(const uint8_t* text, int len) {
  for (int i = 0; i < len + 4; i++) {
    uint8_t disp[4] = {0, 0, 0, 0};
    for (int j = 0; j < 4; j++) {
      int idx = i - j;
      if (idx >= 0 && idx < len) disp[3 - j] = text[idx];
    }
    display.setSegments(disp);
    delay(300);
  }
}
```
