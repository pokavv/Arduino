# HC-05 블루투스 2.0 SPP 모듈

## 개요

HC-05는 Jinan Wavesen Electronics에서 제조한 Bluetooth 2.0 Classic SPP(Serial Port Profile) 모듈입니다.
2.4GHz 대역에서 동작하며, UART 직렬 통신을 무선으로 대체하는 가장 간단하고 저렴한 방법을 제공합니다.

HC-05의 가장 큰 특징은 **마스터/슬레이브 모드를 모두 지원**한다는 점입니다.
AT 명령어로 모드를 전환하여 두 Arduino 간 무선 통신, 스마트폰-Arduino 통신 등에 활용됩니다.

Arduino 프로젝트에서의 활용:
- 스마트폰 앱 ↔ Arduino 무선 제어 (RC카, 로봇 팔 등)
- Arduino ↔ Arduino 무선 통신 (마스터/슬레이브 페어링)
- PC 블루투스 동글 ↔ Arduino 시리얼 모니터 대체
- 블루투스 원격 데이터 로깅

## 핀 구성

### HC-05 모듈 (브레이크아웃 보드 기준, 6핀)

```
[ KEY ] [ VCC ] [ GND ] [ TXD ] [ RXD ] [ STATE ]
  핀1     핀2     핀3     핀4     핀5      핀6
```

| 핀 | 이름 | 방향 | 기능 |
|----|------|------|------|
| 1 | KEY (EN) | 입력 | AT 명령 모드 진입 제어. 전원 인가 전 HIGH → AT 명령 모드(38400bps). 일반 모드는 연결 불필요 또는 LOW |
| 2 | VCC | 전원 | 전원 입력. 3.3V~6V (모듈 내 3.3V 레귤레이터 내장) |
| 3 | GND | 전원 | 공통 접지 |
| 4 | TXD | 출력 | UART 데이터 송신. Arduino RX에 연결. 출력 레벨: 3.3V |
| 5 | RXD | 입력 | UART 데이터 수신. Arduino TX에 연결. 입력 내성: 3.3V (5V 직결 시 장기적 손상 가능) |
| 6 | STATE | 출력 | 블루투스 연결 상태 표시. 연결 중: LOW+고속 깜박임 / 연결됨: HIGH (또는 저속 깜박임 모델 존재) |

> HC-05 베어 모듈(SMD)은 34핀이지만, 브레이크아웃 보드 형태로 판매되는 6핀 버전이 일반적입니다.

## 동작 원리

HC-05는 내부적으로 Cambridge Silicon Radio(CSR)의 BC417 칩을 기반으로 합니다.

**두 가지 동작 모드**:
1. **데이터 모드**: 일반 통신 상태. UART로 주고받는 데이터가 그대로 블루투스로 전송됨 (투명 직렬 통신)
2. **AT 명령 모드**: 설정 변경 상태. 모듈에 AT 명령어를 보내 이름, 비밀번호, 전송 속도 등 설정

**SPP(Serial Port Profile)란**:
블루투스 위에 가상 직렬 포트(COM 포트)를 구현한 프로파일.
PC/스마트폰에서 블루투스 페어링 후 가상 COM 포트로 인식 → Arduino의 Serial.print()가 스마트폰 앱에서 수신됨.

## 아두이노 연결 방법

### 5V Arduino (Uno, Nano 등)와 연결

```
HC-05 VCC ──── Arduino 5V
HC-05 GND ──── Arduino GND
HC-05 TXD ──── Arduino Pin 10 (SoftwareSerial RX)
HC-05 RXD ──── [1kΩ] ──── Arduino Pin 11 (SoftwareSerial TX)
                      └── [2kΩ] ──── GND
          (분압으로 5V TX → 3.3V RX 레벨 변환)
HC-05 KEY ──── Arduino Pin 9 (AT 모드 진입용, 옵션)
HC-05 STATE ── Arduino Pin 8 (연결 상태 감지, 옵션)
```

> 레벨 변환 이유: Arduino TX는 5V 출력 → HC-05 RXD는 3.3V 허용. 1kΩ/2kΩ 분압 또는 레벨 변환 IC 사용.

### ESP32-C3 (3.3V)와 연결

```
HC-05 VCC ──── ESP32 3V3 (모듈 내장 레귤레이터가 3.3V 수용)
HC-05 GND ──── ESP32 GND
HC-05 TXD ──── ESP32 GPIO20 (RX0) 또는 임의 UART RX 핀
HC-05 RXD ──── ESP32 GPIO21 (TX0) 또는 임의 UART TX 핀
              (3.3V 레벨 일치, 레벨 변환 불필요)
```

## 설정 방법 (AT 명령어)

### AT 명령 모드 진입

**방법 1 (전원 인가 전 KEY HIGH)**:
```
1. HC-05 VCC에 전원 차단
2. KEY 핀을 HIGH(3.3V 또는 5V)로 설정
3. 전원 인가
4. LED가 느리게(약 2초 간격) 깜박이면 AT 모드 성공
5. 38400bps로 통신
```

**방법 2 (전원 인가 후 KEY 버튼 누름)**:
```
일부 모듈은 KEY 버튼이 있어 전원 켜진 상태에서 버튼 누르면 AT 모드 진입
→ 이 경우 현재 UART 속도로 통신
```

### 주요 AT 명령어 목록

| 명령어 | 설명 | 응답 |
|--------|------|------|
| `AT` | 통신 확인 | `OK` |
| `AT+VERSION?` | 펌웨어 버전 조회 | `+VERSION:2.0-20100601` |
| `AT+NAME?` | 현재 이름 조회 | `+NAME:HC-05` |
| `AT+NAME=MyBT` | 이름 변경 | `OK` |
| `AT+PSWD?` | 현재 PIN 조회 | `+PSWD:1234` |
| `AT+PSWD=0000` | PIN 변경 | `OK` |
| `AT+UART?` | UART 속도 조회 | `+UART:9600,0,0` |
| `AT+UART=115200,0,0` | UART 속도 변경 (115200, 정지1, 패리티0) | `OK` |
| `AT+ROLE?` | 역할 조회 (0=슬레이브, 1=마스터) | `+ROLE:0` |
| `AT+ROLE=1` | 마스터 모드로 변경 | `OK` |
| `AT+ROLE=0` | 슬레이브 모드로 변경 | `OK` |
| `AT+ADDR?` | 모듈 블루투스 주소 조회 | `+ADDR:1234:56:ABCDEF` |
| `AT+BIND=1234,56,ABCDEF` | 마스터로 특정 주소에 바인딩 | `OK` |
| `AT+RESET` | 재시작 | `OK` |
| `AT+ORGL` | 공장 초기화 | `OK` |

### AT 명령어 사용 예 (Arduino 코드)

```cpp
#include <SoftwareSerial.h>

SoftwareSerial bt(10, 11);  // RX=10, TX=11

void setup() {
    Serial.begin(9600);   // 시리얼 모니터용
    bt.begin(38400);      // AT 모드 기본 속도
    Serial.println("AT 명령 모드. 명령어를 입력하세요.");
}

void loop() {
    // 시리얼 모니터 → HC-05
    if (Serial.available()) {
        bt.write(Serial.read());
    }
    // HC-05 → 시리얼 모니터
    if (bt.available()) {
        Serial.write(bt.read());
    }
}
```

### 기본 데이터 송수신 코드

```cpp
#include <SoftwareSerial.h>

SoftwareSerial bt(10, 11);  // RX=10, TX=11

void setup() {
    Serial.begin(9600);
    bt.begin(9600);  // 설정한 UART 속도와 일치시킬 것
    Serial.println("HC-05 준비 완료");
}

void loop() {
    // 스마트폰에서 받은 데이터 처리
    if (bt.available()) {
        char cmd = bt.read();
        Serial.print("수신: ");
        Serial.println(cmd);
        if (cmd == '1') {
            digitalWrite(LED_BUILTIN, HIGH);
            bt.println("LED ON");
        } else if (cmd == '0') {
            digitalWrite(LED_BUILTIN, LOW);
            bt.println("LED OFF");
        }
    }
}
```

## 주의사항

1. **5V 레벨 변환 필수**: 5V Arduino의 TX → HC-05 RXD에 직결 시 장기적으로 모듈이 손상됩니다. 분압 저항 또는 레벨 변환 IC 사용.
2. **충분한 전류 공급**: HC-05는 페어링 시 최대 40mA를 소비합니다. Arduino 3.3V 핀(최대 50mA)에서는 빠듯합니다. 5V 핀에서 공급 권장.
3. **SoftwareSerial 속도 한계**: Arduino Uno의 SoftwareSerial은 115200bps 이상에서 불안정합니다. 9600 또는 38400bps 권장.
4. **동시 사용 불가**: 한 번에 하나의 기기와만 연결됩니다. 다수 기기 동시 연결은 불가.
5. **iOS 미지원**: HC-05는 Bluetooth Classic SPP 프로파일. iOS에서는 SPP를 지원하지 않습니다. iOS는 HM-10(BLE) 사용.

### HC-05 두 개 마스터/슬레이브 페어링

```
모듈 A (마스터 설정):
  AT+ROLE=1
  AT+CMODE=0         (특정 주소에만 연결)
  AT+BIND=모듈B주소

모듈 B (슬레이브 설정, 기본):
  AT+ROLE=0
  AT+ADDR? → 주소 확인 후 모듈 A에 입력

전원 켜면 자동으로 마스터가 슬레이브를 찾아 연결
→ Arduino A ↔ HC-05 마스터 ~~블루투스~~ HC-05 슬레이브 ↔ Arduino B
→ 양쪽 Arduino에서 Serial.print() → 상대방에서 Serial.read()로 수신
```
