# 7-세그먼트 디스플레이

## 개요

7-세그먼트 디스플레이(Seven-Segment Display)는 7개의 LED 세그먼트(a~g)와 소수점(dp) 세그먼트로 구성된 숫자 표시 장치입니다.  
각 세그먼트의 ON/OFF 조합으로 0~9 숫자, 일부 영문자(A, b, C, d, E, F, H, L, P 등)를 표시할 수 있습니다.  
아두이노에서는 온도 표시, 카운터, 타이머, 점수 표시판, 간단한 숫자 UI 등에 활용됩니다.

---

## 종류 및 모델 비교

### 공통 음극 vs 공통 양극

| 항목 | 공통 음극 (CC) | 공통 양극 (CA) |
|------|--------------|--------------|
| 공통 핀 | GND에 연결 | VCC에 연결 |
| 세그먼트 켜는 방법 | 해당 핀에 HIGH | 해당 핀에 LOW |
| 제어 극성 | Active HIGH | Active LOW |
| 주요 모델 | 5161AS, LTS-546AP | 5161BS, LTS-546AK |

### 크기 및 자릿수별 종류

| 종류 | 자릿수 | 표시 면적 | 특징 |
|------|--------|---------|------|
| 0.36인치 단일 | 1자리 | 소형 | 핀 10개, 브레드보드 적합 |
| 0.56인치 단일 | 1자리 | 표준 | 핀 10개, 가장 일반적 |
| 0.56인치 4자리 | 4자리 | 표준 | 핀 12개, 멀티플렉싱 필요 |
| 1.0인치 단일 | 1자리 | 대형 | 원거리 가시성 |
| TM1637 모듈 | 4자리 | 표준 | I2C, 클럭 표시용 |
| MAX7219 모듈 | 8자리 | 표준 | SPI, 8개 제어 가능 |

## 핀 구성

### 단일 1자리 (10핀, 0.56인치 기준)

```
        ┌───────────────┐
   1 ───┤ e    (상단)  f ├─── 6  ← 상단 윗면 핀
   2 ───┤ d            g ├─── 7
   3 ───┤ dp          a  ├─── 8
   4 ───┤ c          CC1 ├─── 9  ← 공통 핀 1
   5 ───┤ CC2         b  ├─── 10 ← 공통 핀 2
        └───────────────┘

    세그먼트 배치:
         aaa
        f   b
        f   b
         ggg
        e   c
        e   c
         ddd  .dp
```

| 핀 번호 | 기능 (CC 모델) | 세그먼트 위치 |
|---------|--------------|------------|
| 1 | e | 왼쪽 아래 수직 |
| 2 | d | 아래 수평 |
| 3 | dp | 소수점 |
| 4 | c | 오른쪽 아래 수직 |
| 5 | CC2 (GND) | 공통 음극 2 |
| 6 | f | 왼쪽 위 수직 |
| 7 | g | 중간 수평 |
| 8 | a | 위쪽 수평 |
| 9 | CC1 (GND) | 공통 음극 1 |
| 10 | b | 오른쪽 위 수직 |

## 동작 원리

7-세그먼트 디스플레이는 내부적으로 LED 어레이입니다.  
각 세그먼트(a~g, dp)를 독립적으로 켜고 끄는 조합으로 숫자를 표현합니다.

### 숫자별 세그먼트 ON 표

| 숫자 | a | b | c | d | e | f | g |
|------|---|---|---|---|---|---|---|
| 0 | O | O | O | O | O | O | X |
| 1 | X | O | O | X | X | X | X |
| 2 | O | O | X | O | O | X | O |
| 3 | O | O | O | O | X | X | O |
| 4 | X | O | O | X | X | O | O |
| 5 | O | X | O | O | X | O | O |
| 6 | O | X | O | O | O | O | O |
| 7 | O | O | O | X | X | X | X |
| 8 | O | O | O | O | O | O | O |
| 9 | O | O | O | O | X | O | O |

### 4자리 멀티플렉싱

4자리 디스플레이는 핀 수를 줄이기 위해 세그먼트 핀을 공유합니다.  
각 자리(D1~D4)를 빠르게 순서대로 켜면서 해당 자리의 숫자를 표시합니다.  
사람 눈의 잔상 효과(> 50Hz)로 모든 자리가 동시에 켜진 것처럼 보입니다.

```
시간  t1     t2     t3     t4
자리  D1 ON  D2 ON  D3 ON  D4 ON
세그  1234   5678   3456   9012
→ 눈에는 "1234 5678 3456 9012" 모두 켜진 것처럼 인식
```

## 아두이노 연결 방법

### 저항 계산 (각 세그먼트당)

```
R = (Vcc - Vf) / If

예시 (5V, Vf=2.0V, If=10mA):
  R = (5.0 - 2.0) / 0.010 = 300Ω → 330Ω 사용

예시 (3.3V, Vf=2.0V, If=10mA):
  R = (3.3 - 2.0) / 0.010 = 130Ω → 150Ω 사용
```

### 직접 GPIO 연결 (1자리, 공통 음극)

```
GPIO_a ──[150Ω]── 핀8(a)
GPIO_b ──[150Ω]── 핀10(b)
GPIO_c ──[150Ω]── 핀4(c)
GPIO_d ──[150Ω]── 핀2(d)
GPIO_e ──[150Ω]── 핀1(e)
GPIO_f ──[150Ω]── 핀6(f)
GPIO_g ──[150Ω]── 핀7(g)
핀5, 핀9 (CC) ──── GND
```
> 1자리에 GPIO 7~8개가 필요합니다. 핀 절약을 위해 IC(74HC595, TM1637, MAX7219) 사용 권장.

### 시프트 레지스터 74HC595 사용 (GPIO 3개로 8비트 제어)

```
ESP32 GPIO(SER/DATA) → 74HC595 DS (핀14)
ESP32 GPIO(SRCLK/CLK) → 74HC595 SHCP (핀11)
ESP32 GPIO(RCLK/LATCH) → 74HC595 STCP (핀12)
74HC595 Q0~Q7 → 세그먼트 a~g, dp (각각 150Ω 직렬)
74HC595 OE (핀13) → GND
74HC595 MR (핀10) → VCC
```

## 코드 예제 개념

### 직접 GPIO 제어 (1자리, 공통 음극)

```cpp
// 세그먼트 핀 배열 [a, b, c, d, e, f, g]
const int SEG_PINS[7] = {2, 3, 4, 5, 6, 7, 8};

// 숫자별 세그먼트 패턴 (a b c d e f g)
// 1 = 켜짐, 0 = 꺼짐
const uint8_t DIGITS[10] = {
    0b1111110,  // 0: a,b,c,d,e,f ON
    0b0110000,  // 1: b,c ON
    0b1101101,  // 2: a,b,d,e,g ON
    0b1111001,  // 3: a,b,c,d,g ON
    0b0110011,  // 4: b,c,f,g ON
    0b1011011,  // 5: a,c,d,f,g ON
    0b1011111,  // 6: a,c,d,e,f,g ON
    0b1110000,  // 7: a,b,c ON
    0b1111111,  // 8: 전체 ON
    0b1111011,  // 9: a,b,c,d,f,g ON
};

void setup() {
    for (int i = 0; i < 7; i++) {
        pinMode(SEG_PINS[i], OUTPUT);
    }
}

void displayDigit(int num) {
    for (int i = 0; i < 7; i++) {
        // MSB(비트6)가 a, LSB(비트0)가 g
        bool segOn = (DIGITS[num] >> (6 - i)) & 1;
        digitalWrite(SEG_PINS[i], segOn ? HIGH : LOW);
    }
}

void loop() {
    for (int i = 0; i <= 9; i++) {
        displayDigit(i);
        delay(500);
    }
}
```

### TM1637 모듈 사용 (4자리, I2C 유사 프로토콜)

```cpp
#include <TM1637Display.h>

const int CLK_PIN = 4;
const int DIO_PIN = 5;

TM1637Display display(CLK_PIN, DIO_PIN);

void setup() {
    display.setBrightness(7); // 0~7 밝기
}

void loop() {
    display.showNumberDec(1234);      // 1234 표시
    delay(1000);
    display.showNumberDecEx(1234, 0b01000000, true); // 12:34 (콜론 포함)
    delay(1000);
    display.clear();                  // 화면 지우기
    delay(500);
}
```

### MAX7219 사용 (SPI, 8자리 연속 제어)

```cpp
#include <LedControl.h>

// LedControl(DIN, CLK, CS, 디바이스수)
LedControl lc = LedControl(11, 13, 10, 1);

void setup() {
    lc.shutdown(0, false);   // 절전 모드 해제
    lc.setIntensity(0, 8);   // 밝기 0~15
    lc.clearDisplay(0);
}

void loop() {
    lc.setDigit(0, 7, 1, false);  // 디바이스0, 자리7, 숫자1, 소수점X
    lc.setDigit(0, 6, 2, false);
    lc.setDigit(0, 5, 3, true);   // 소수점 표시
    lc.setDigit(0, 4, 4, false);
}
```

## 마킹/식별 방법

| 확인 방법 | 설명 |
|---------|------|
| 부품 번호 마지막 자리 | AS = 공통 음극, BS = 공통 양극 (일반적 규칙, 제조사마다 다름) |
| 멀티미터 다이오드 모드 | 공통 핀 후보에 -단자, 나머지에 +단자 → 숫자 뜨면 CC |
| 데이터시트 | 구매처 제품 페이지 또는 부품 번호로 검색 |
| 직접 테스트 | 3V 전원 + 330Ω 저항으로 핀을 하나씩 탐침 |

## 주의사항

1. **개별 저항 필수**: 각 세그먼트에 직렬 저항이 필요합니다. 저항 없이 연결 시 즉시 손상.
2. **CC/CA 혼동 금지**: 잘못 연결 시 동작 안 하거나 LED 손상. 구매 시 반드시 확인.
3. **GPIO 핀 수 절약**: 직접 연결은 GPIO 7~8개가 필요하므로 핀이 부족한 경우 74HC595, TM1637, MAX7219 사용.
4. **멀티플렉싱 타이밍**: 4자리 직접 제어 시 각 자리 표시 시간이 최소 2ms 이상이어야 깜빡임 없음. 총 반복 주기 < 20ms 유지.
5. **전류 합산**: 8개 세그먼트 전체 ON 시 약 160mA (10mA × 8 + dp). 아두이노 5V 핀 직접 공급 한계에 주의.
6. **ESP32-C3 3.3V**: Vf=2.0V의 세그먼트에 3.3V 전원 사용 시 150Ω 저항 사용.

## 자주 쓰이는 회로 패턴

### 패턴 1: 1자리 직접 GPIO 연결

```
GPIO × 7개 ──[150Ω 각각]── 세그먼트 a~g
공통 핀 ──── GND (CC)
```

### 패턴 2: 74HC595 + 1자리

```
GPIO × 3개 (DATA, CLK, LATCH) → 74HC595 → 세그먼트 8개 (a~g, dp)
→ GPIO 3개로 1자리 전체 제어
```

### 패턴 3: TM1637 모듈 (4자리 클럭 디스플레이)

```
GPIO × 2개 (CLK, DIO) → TM1637 → 4자리 세그먼트
→ 라이브러리로 숫자, 콜론, 밝기 제어
```

### 패턴 4: MAX7219 + 8자리 (다중 디스플레이 연결 가능)

```
GPIO × 3개 (DIN, CLK, CS) → MAX7219 → 8자리 세그먼트
→ 여러 MAX7219를 데이지 체인으로 연결해 64자리까지 확장
```
