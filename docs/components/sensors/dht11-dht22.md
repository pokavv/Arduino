# DHT11 / DHT22 — 온습도 센서

## 개요

DHT11과 DHT22는 AOSONG(Guangzhou Aosong Electronics)에서 제조한 단선 디지털 온습도 센서입니다.
내부에 용량식(Capacitive) 습도 감지 소자와 NTC(Negative Temperature Coefficient) 서미스터를 내장하고,
전용 8비트 마이크로컨트롤러가 신호를 처리하여 단일 데이터 핀으로 디지털 값을 출력합니다.

**측정 원리**
- 습도: 수분 함유량에 따라 용량이 변하는 폴리머 커패시터로 측정
- 온도: 저항값이 온도에 따라 변하는 NTC 서미스터로 측정
- 내장 ADC가 아날로그 신호를 디지털로 변환 후 단선 프로토콜로 전송

**아두이노에서의 활용**
환경 모니터링, 스마트 홈, 온실 자동화, 날씨 스테이션 등에 폭넓게 활용됩니다.
`DHT` 라이브러리(Adafruit) 또는 `SimpleDHT` 라이브러리로 간편하게 사용합니다.

| 샘플링 속도 | 1회/초 (1 Hz) | 0.5회/초 (0.5 Hz) |

> 야외·정밀 제어 필요 시 DHT22를 선택합니다.

## 핀 구성

```
DHT11 / DHT22 (전면에서 봤을 때, 그물망이 앞)

  ┌──────────┐
  │  1  2  3  4  │   (핀 번호: 왼쪽 → 오른쪽)
  └──────────┘
     │  │  │  │
    VCC DATA NC GND
```

| 핀 번호 | 이름 | 역할 |
|---------|------|------|
| 1 | VCC | 전원 공급 (+3.3V ~ 5V) |
| 2 | DATA | 단선 디지털 데이터 입출력 |
| 3 | NC | 미연결 (Not Connected) |
| 4 | GND | 접지 (0V) |

> 모듈 형태(3핀 보드) 구매 시 핀 배열이 다를 수 있으므로 실크스크린 확인 필수.

## 통신 프로토콜 — 단선 직렬 (Single-Wire Serial)

### 프로토콜 개요

DHT 센서는 독자적인 단선 비동기 직렬 프로토콜을 사용합니다.
MCU가 마스터, DHT가 슬레이브 역할을 합니다.

### 통신 순서

```
1. MCU → DHT : 시작 신호 (Start Signal)
2. DHT → MCU : 응답 신호 (Response Signal)
3. DHT → MCU : 40비트 데이터 전송
```

### 1단계: 시작 신호

```
MCU가 DATA 핀을 OUTPUT으로 설정:
  - LOW 출력: DHT11 최소 18ms, DHT22 최소 1ms (권장 20ms)
  - HIGH 출력: 20 ~ 40 µs 대기
  - 이후 DATA 핀을 INPUT으로 전환
```

### 2단계: 응답 신호

```
DHT가 응답:
  - LOW: 80 µs
  - HIGH: 80 µs
  (이후 데이터 전송 시작)
```

### 3단계: 데이터 비트 인코딩

```
각 비트는 LOW 50µs로 시작 후:
  - '0' 비트: HIGH 26 ~ 28 µs
  - '1' 비트: HIGH 70 µs
```

### 40비트 데이터 구조

```
[8비트 습도 정수] [8비트 습도 소수] [8비트 온도 정수] [8비트 온도 소수] [8비트 체크섬]

DHT11:
  - 습도 정수: 0~100 (소수부는 항상 0)
  - 온도 정수: 0~50 (소수부는 항상 0)

DHT22:
  - 습도 = (정수부 × 256 + 소수부) / 10.0
  - 온도 정수 비트15가 1이면 음수
  - 온도 = (정수부 & 0x7F × 256 + 소수부) / 10.0
```

### 체크섬 검증

```
체크섬 = (습도 정수 + 습도 소수 + 온도 정수 + 온도 소수) & 0xFF
수신된 체크섬과 일치하면 데이터 유효
```

## 아두이노 연결 방법

### 회로 구성

```
ESP32-C3          DHT11/DHT22
  3.3V ──────────── VCC (1번)
  GND  ──────────── GND (4번)
  G4   ──┬───────── DATA (2번)
         │
        4.7kΩ (풀업 저항)
         │
  3.3V ──┘
```

> **풀업 저항 필수**: DATA 핀과 VCC 사이에 반드시 4.7kΩ (또는 10kΩ) 저항을 연결합니다.
> 일부 모듈 보드에는 내장되어 있으므로 확인 후 중복 연결을 피하세요.

### 전원 안정화 커패시터

```
VCC와 GND 사이에 100nF (0.1µF) 바이패스 커패시터를 센서 가까이 배치하면
노이즈로 인한 오류를 줄일 수 있습니다.
```

### 라이브러리 설치

Arduino IDE > 라이브러리 매니저에서:
- `DHT sensor library` by Adafruit 검색 후 설치
- 의존성: `Adafruit Unified Sensor` 함께 설치

### 기본 코드 예시

```cpp
#include <DHT.h>

#define DHT_PIN   4       // DATA 핀 연결 GPIO
#define DHT_TYPE  DHT22   // 또는 DHT11

DHT dht(DHT_PIN, DHT_TYPE);

void setup() {
  Serial.begin(115200);
  dht.begin();
  delay(2000); // 센서 안정화 대기
}

void loop() {
  // 읽기 간격: DHT11 최소 1초, DHT22 최소 2초
  float humidity    = dht.readHumidity();
  float temperature = dht.readTemperature(); // 섭씨

  if (isnan(humidity) || isnan(temperature)) {
    Serial.println("센서 읽기 실패!");
    return;
  }

  // 열지수 계산 (선택)
  float heatIndex = dht.computeHeatIndex(temperature, humidity, false);

  Serial.printf("온도: %.1f°C  습도: %.1f%%  열지수: %.1f°C\n",
                temperature, humidity, heatIndex);

  delay(2000);
}
```

## 측정값 계산 공식

### 절대 습도 (g/m³)

```
절대습도 = (6.112 × e^(17.67×T / (T+243.5)) × RH × 2.1674) / (273.15 + T)
  T  = 온도 (°C)
  RH = 상대 습도 (%)
```

### 이슬점 온도 (Dew Point, °C)

```
γ = ln(RH/100) + (17.62 × T) / (243.12 + T)
이슬점 = 243.12 × γ / (17.62 - γ)
```

### 열지수 (Heat Index, °C) — Steadman 공식 (Adafruit 라이브러리 내장)

```
HI = -8.784695 + 1.61139411×T + 2.3385249×RH - 0.14611605×T×RH
     - 0.01230809×T² - 0.01642482×RH² + 0.00221173×T²×RH
     + 0.00072546×T×RH² - 0.00000358×T²×RH²
```

## 주의사항

1. **샘플링 간격 엄수**
   - DHT11: 두 번의 읽기 사이 최소 **1초** 이상
   - DHT22: 두 번의 읽기 사이 최소 **2초** 이상
   - 간격을 지키지 않으면 NaN(읽기 실패) 또는 이전 값이 반환됩니다.

2. **풀업 저항 필수**
   - 풀업 저항 없이 연결하면 통신 오류 발생
   - 권장값: 4.7kΩ (데이터선 길이가 20m 이상이면 1kΩ으로 낮춤)

3. **전원 투입 후 안정화 대기**
   - 전원 인가 후 최소 1~2초 대기 후 읽기 시작
   - `setup()` 내 `delay(2000)` 권장

4. **ESP32-C3 전압 주의**
   - ESP32-C3의 GPIO는 3.3V 레벨
   - 센서를 5V로 공급해도 DATA 핀 레벨은 3.3V 이하로 유지 필요
   - 3.3V 공급 시 신호 레벨 문제 없음 (단, 측정 범위가 약간 줄어들 수 있음)

5. **결로 환경 금지**
   - 습도 100% 환경이나 결로가 생기는 조건에서 장기 사용은 센서 손상 가능

6. **선 길이**
   - 데이터 케이블이 20m 이상이면 버퍼 IC 또는 풀업 저항 값 조정 필요

## 캘리브레이션 방법

DHT11/DHT22는 공장 출하 시 캘리브레이션이 완료된 제품이므로 사용자 캘리브레이션은 기본적으로 불필요합니다.
그러나 정밀 적용 시 다음 방법으로 오프셋 보정을 적용할 수 있습니다.

### 오프셋 보정 방법

```cpp
// 기준 온도계(±0.1°C급)와 비교하여 오프셋 측정
const float TEMP_OFFSET = -0.5;  // 예: 측정값이 기준보다 0.5°C 높으면 -0.5
const float HUMI_OFFSET =  2.0;  // 예: 측정값이 기준보다 2% 낮으면 +2.0

float temperature = dht.readTemperature() + TEMP_OFFSET;
float humidity    = dht.readHumidity()    + HUMI_OFFSET;
```

### 이동평균 필터 (노이즈 감소)

```cpp
const int SAMPLE_COUNT = 5;
float tempSamples[SAMPLE_COUNT] = {0};
int   sampleIndex = 0;

float getAverageTemp() {
  tempSamples[sampleIndex] = dht.readTemperature();
  sampleIndex = (sampleIndex + 1) % SAMPLE_COUNT;

  float sum = 0;
  for (int i = 0; i < SAMPLE_COUNT; i++) sum += tempSamples[i];
  return sum / SAMPLE_COUNT;
}
```

## 실사용 팁

- **NaN 처리 루틴을 반드시 작성**: `isnan()` 검사 후 이전 유효값을 사용하는 방어 코드 권장
- **millis() 기반 비동기 읽기**: `delay()` 대신 millis()로 간격을 제어하면 다른 작업과 병행 가능
- **습도 100% 포화 방지**: 화분, 수족관 근처에 사용 시 방수 케이스 또는 DS18B20(온도 전용) 조합 고려
- **배터리 절전**: `대기 전류`가 DHT22는 40µA로 낮아 배터리 프로젝트에 적합; 측정 전에만 전원 공급도 가능
- **직사광선 차단**: 직사광선이 닿으면 온도가 높게 측정됨 — 반사 차단 덮개 필요
- **여러 센서 사용**: 각 센서에 개별 GPIO 핀 및 풀업 저항 필요 (버스 공유 불가)
