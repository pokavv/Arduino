# RGB LED

## 개요

RGB LED는 빨강(Red), 녹색(Green), 파랑(Blue) 세 가지 LED를 하나의 패키지에 통합한 소자입니다.  
세 채널의 밝기를 독립적으로 제어하여 이론상 1,670만 가지 색상을 표현할 수 있습니다.  
아두이노에서는 상태 표시, 분위기 조명, 색상 피드백, UI 인디케이터 등에 활용됩니다.

**공통 음극 (Common Cathode, CC)**: 세 LED의 캐소드가 공통 GND에 연결된 구조.  
**공통 양극 (Common Anode, CA)**: 세 LED의 애노드가 공통 VCC에 연결된 구조.

## 핀 구성

### 5mm 관통형 (4핀)

```
정면에서 바라볼 때 핀 번호 (왼쪽부터):

    1    2    3    4
    |    |    |    |
   [R]  [공통][G]  [B]

공통 음극 (CC):
  핀1 = R (빨강 애노드)
  핀2 = GND (공통 캐소드) ← 가장 긴 핀
  핀3 = G (녹색 애노드)
  핀4 = B (파랑 애노드)

공통 양극 (CA):
  핀1 = R (빨강 캐소드)
  핀2 = VCC (공통 애노드) ← 가장 긴 핀
  핀3 = G (녹색 캐소드)
  핀4 = B (파랑 캐소드)
```

| 핀 번호 | CC 기능 | CA 기능 | 설명 |
|---------|---------|---------|------|
| 1 | 빨강 (R) 애노드 | 빨강 (R) 캐소드 | 빨강 채널 제어 |
| 2 | 공통 캐소드 (GND) | 공통 애노드 (VCC) | **가장 긴 핀** |
| 3 | 녹색 (G) 애노드 | 녹색 (G) 캐소드 | 녹색 채널 제어 |
| 4 | 파랑 (B) 애노드 | 파랑 (B) 캐소드 | 파랑 채널 제어 |

## 동작 원리

RGB LED는 세 개의 독립적인 LED가 하나의 패키지에 들어있습니다.  
각 채널의 전류량(또는 PWM 듀티)을 조절하여 색 혼합을 구현합니다.

**빛의 삼원색 혼합 (가산혼합):**
- R + G = 노랑 (Yellow)
- G + B = 시안 (Cyan)
- R + B = 마젠타 (Magenta)
- R + G + B = 흰색 (White)
- 모두 OFF = 검정 (Black)

**PWM 원리:**  
각 채널에 PWM 신호(0~255)를 입력하여 평균 전류를 조절합니다.  
눈은 빠른 점멸을 밝기로 인식하므로 듀티 50% = 절반 밝기로 보입니다.

## 아두이노 연결 방법

### 저항 계산

각 채널에 독립적인 직렬 저항이 필요합니다 (채널별 Vf가 다름).

```
R = (Vcc - Vf) / If

채널별 예시 (Vcc = 3.3V, If = 10mA):
  빨강: (3.3 - 2.0) / 0.010 = 130Ω → 150Ω
  녹색: (3.3 - 3.2) / 0.010 = 10Ω  → 33Ω (최소)
  파랑: (3.3 - 3.2) / 0.010 = 10Ω  → 33Ω (최소)

채널별 예시 (Vcc = 5V, If = 10mA):
  빨강: (5.0 - 2.0) / 0.010 = 300Ω → 330Ω
  녹색: (5.0 - 3.2) / 0.010 = 180Ω → 180Ω
  파랑: (5.0 - 3.2) / 0.010 = 180Ω → 180Ω
```

### 공통 음극 배선

```
GPIO_R ──[150Ω]── 핀1(R)
                  핀2(GND) ──── GND
GPIO_G ──[33Ω]── 핀3(G)
GPIO_B ──[33Ω]── 핀4(B)
```

### 공통 양극 배선

```
                  핀2(VCC) ──── 3.3V
GPIO_R ──[150Ω]── 핀1(R)
GPIO_G ──[33Ω]── 핀3(G)
GPIO_B ──[33Ω]── 핀4(B)
```
> 공통 양극은 GPIO를 LOW로 설정해야 해당 채널이 켜집니다 (Active LOW).

## 코드 예제 개념

### 공통 음극 (CC) — Active HIGH

```cpp
// 핀 정의
const int PIN_R = 4;
const int PIN_G = 5;
const int PIN_B = 6;

// PWM 설정 (ESP32-C3 전용)
const int PWM_FREQ = 5000;
const int PWM_BITS = 8;

void setup() {
    // 각 채널에 LEDC PWM 설정
    ledcSetup(0, PWM_FREQ, PWM_BITS);  // 채널0 = 빨강
    ledcSetup(1, PWM_FREQ, PWM_BITS);  // 채널1 = 녹색
    ledcSetup(2, PWM_FREQ, PWM_BITS);  // 채널2 = 파랑
    ledcAttachPin(PIN_R, 0);
    ledcAttachPin(PIN_G, 1);
    ledcAttachPin(PIN_B, 2);
}

// 색상 설정 함수 (0~255 범위)
void setColor(uint8_t r, uint8_t g, uint8_t b) {
    ledcWrite(0, r);
    ledcWrite(1, g);
    ledcWrite(2, b);
}

void loop() {
    setColor(255, 0, 0);    // 빨강
    delay(1000);
    setColor(0, 255, 0);    // 녹색
    delay(1000);
    setColor(0, 0, 255);    // 파랑
    delay(1000);
    setColor(255, 255, 0);  // 노랑
    delay(1000);
    setColor(0, 255, 255);  // 시안
    delay(1000);
    setColor(255, 0, 255);  // 마젠타
    delay(1000);
    setColor(255, 255, 255); // 흰색
    delay(1000);
}
```

### 공통 양극 (CA) — Active LOW

```cpp
// 공통 양극은 값을 반전시킴
void setColorCA(uint8_t r, uint8_t g, uint8_t b) {
    ledcWrite(0, 255 - r);  // 반전
    ledcWrite(1, 255 - g);
    ledcWrite(2, 255 - b);
}
```

### 무지개 색상 순환 (HSV → RGB 변환)

```cpp
// HSV(색상, 채도, 명도) → RGB 변환
// hue: 0~360, sat: 0~255, val: 0~255
void hsvToRgb(int hue, uint8_t sat, uint8_t val,
              uint8_t &r, uint8_t &g, uint8_t &b) {
    uint8_t region = hue / 60;
    uint8_t remainder = (hue % 60) * 4;
    uint8_t p = (val * (255 - sat)) >> 8;
    uint8_t q = (val * (255 - ((sat * remainder) >> 8))) >> 8;
    uint8_t t = (val * (255 - ((sat * (255 - remainder)) >> 8))) >> 8;
    switch (region) {
        case 0: r=val; g=t;   b=p;   break;
        case 1: r=q;   g=val; b=p;   break;
        case 2: r=p;   g=val; b=t;   break;
        case 3: r=p;   g=q;   b=val; break;
        case 4: r=t;   g=p;   b=val; break;
        default:r=val; g=p;   b=q;   break;
    }
}
```

## 마킹/식별 방법

| 확인 방법 | 설명 |
|---------|------|
| 가장 긴 핀 | 공통 핀 (CC이면 GND, CA이면 VCC) |
| 멀티미터 다이오드 모드 | 공통 핀을 기준으로 각 핀을 탐침하면 순방향 전압 확인 가능 |
| 데이터시트 참조 | 구매처 제품 페이지에서 CC/CA 여부 확인 필수 |
| 부품 각인 | SMD 타입은 실크스크린에 CC/CA 표기 |

## 주의사항

1. **채널별 개별 저항 필수**: 빨강(Vf≈2V)과 녹색/파랑(Vf≈3.2V)의 순방향 전압이 다르므로 저항값이 달라야 합니다.
2. **CC/CA 혼동 금지**: 공통 음극과 공통 양극을 바꿔 연결하면 동작하지 않거나 LED가 손상됩니다.
3. **GPIO 전류 한계**: ESP32-C3 핀당 최대 40mA. 세 채널 합산 전류가 핀 한계를 초과하지 않도록 합니다.
4. **ESP32-C3 3.3V**: 녹색/파랑 채널의 Vf가 3.2V에 근접하므로 저항을 너무 크게 잡으면 전류가 거의 흐르지 않아 어둡습니다.
5. **색 보정**: 녹색 채널은 눈의 민감도가 높아 같은 전류에서도 더 밝게 보입니다. 흰색 표현 시 녹색 값을 줄여 조정합니다 (예: R=255, G=180, B=200).
6. **PWM 주파수**: 너무 낮은 PWM 주파수(< 50Hz)는 육안으로 깜빡임이 보입니다. 1kHz 이상 권장.

## 자주 쓰이는 회로 패턴

### 패턴 1: 직접 GPIO 구동 (3.3V 시스템, 공통 음극)

```
GPIO4 ──[150Ω]── R핀
GPIO5 ──[33Ω]──  G핀
GPIO6 ──[33Ω]──  B핀
                 공통핀 ──── GND
```

### 패턴 2: 트랜지스터 구동 (5V 전원 사용 시)

```
GPIO ──[1kΩ]── NPN 베이스
               NPN 컬렉터 ──[저항]── LED 채널 ── 5V
               NPN 이미터 ── GND
```
- 5V 전원으로 구동하여 더 밝은 출력 가능.

### 패턴 3: 색상 테이블 기반 상태 표시

```
상태별 색상 매핑:
  정상 → 녹색 (0, 255, 0)
  경고 → 노랑 (255, 200, 0)
  오류 → 빨강 (255, 0, 0)
  대기 → 파랑 (0, 0, 255)
  연결중 → 시안 점멸 (0, 255, 255)
```
