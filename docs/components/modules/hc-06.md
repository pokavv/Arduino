# HC-06 블루투스 슬레이브 전용 모듈

## 개요

HC-06은 HC-05와 같은 제조사(Jinan Wavesen Electronics)의 Bluetooth 2.0 SPP 모듈이지만,
**슬레이브 전용**으로 고정되어 있어 마스터로 전환이 불가능합니다.

대신 AT 명령어 세트가 더 간단하고, 전원 공급 중 언제든 AT 명령 모드로 진입할 수 있어
초보자가 처음 블루투스를 다룰 때 HC-06이 더 쉽습니다.

HC-06의 특징:
- 항상 슬레이브 역할 (스마트폰/PC가 마스터가 되어 연결)
- AT 명령 모드 진입: KEY 핀 없이 전원 켜진 상태에서 미연결 상태면 자동으로 AT 명령 수신
- Arduino ↔ 스마트폰, PC ↔ Arduino 단방향 사용에 최적화
- 더 저렴한 가격

Arduino 프로젝트에서의 활용:
- 스마트폰 앱으로 Arduino 제어 (단방향 슬레이브로 충분한 경우)
- 시리얼 모니터 블루투스 대체
- 단순 센서 데이터 전송

## 모델 비교 표

HC-05 vs HC-06 상세 비교:

| 항목 | HC-05 | HC-06 |
|------|-------|-------|
| 블루투스 역할 | 마스터/슬레이브 전환 | 슬레이브 전용 |
| AT 모드 진입 | KEY HIGH + 전원 ON | 연결 전 미연결 상태 자동 |
| AT 모드 통신 속도 | 38400bps (고정) | 현재 UART 속도 |
| AT 명령어 종류 | 풍부 (BIND, CMODE 등) | 제한적 (이름, PIN, 속도만) |
| 가격 | 조금 비쌈 | 저렴 |
| 사용 난이도 | 중간 | 쉬움 |
| Arduino 간 통신 | 가능 (마스터 설정) | 불가 |
| 스마트폰 연결 | 가능 | 가능 |
| PCB 레이아웃 | 6핀 브레이크아웃 | 4핀 또는 6핀 |

## 핀 구성

### HC-06 모듈 (4핀 브레이크아웃 기준)

```
[ VCC ] [ GND ] [ TXD ] [ RXD ]
  핀1     핀2     핀3     핀4
```

| 핀 | 이름 | 방향 | 기능 |
|----|------|------|------|
| 1 | VCC | 전원 | 전원 입력. 3.3V~6V |
| 2 | GND | 전원 | 공통 접지 |
| 3 | TXD | 출력 | UART 데이터 송신. Arduino RX 핀에 연결. 출력: 3.3V |
| 4 | RXD | 입력 | UART 데이터 수신. Arduino TX 핀에 연결. 입력 내성: 3.3V |

> 6핀 버전은 추가로 KEY, STATE 핀이 있으나 HC-06에서는 KEY가 기능하지 않습니다.

## 동작 원리

HC-06은 기본적으로 슬레이브 역할만 수행합니다.

**동작 흐름**:
```
전원 ON → 블루투스 광고(Advertising) 시작
→ 스마트폰/PC에서 HC-06 검색 및 페어링 요청
→ PIN 코드 입력 (기본: 1234)
→ 연결 완료 → LED 지속 점등 (또는 느린 깜박임)
→ 스마트폰 앱에서 가상 COM 포트로 통신 가능
→ Arduino UART ↔ 블루투스 양방향 투명 전송
```

**AT 명령 모드**:
- HC-06은 전원 ON 후 블루투스 연결이 이루어지지 않은 상태에서 UART로 AT 명령을 받으면 실행
- 별도 AT 모드 진입 절차 없음 (HC-05와 다른 점)
- 연결 중에는 AT 명령 불가 (데이터로 처리됨)

## 아두이노 연결 방법

### 5V Arduino (Uno, Nano 등)와 연결

```
HC-06 VCC ──── Arduino 5V
HC-06 GND ──── Arduino GND
HC-06 TXD ──── Arduino 핀 10 (SoftwareSerial RX)
HC-06 RXD ──── [1kΩ] ──── Arduino 핀 11 (SoftwareSerial TX)
               [2kΩ] ──── GND
          (5V → 3.3V 레벨 분압)
```

### ESP32-C3 (3.3V)와 연결

```
HC-06 VCC ──── ESP32 3V3 또는 5V 핀
HC-06 GND ──── ESP32 GND
HC-06 TXD ──── ESP32 GPIO20 (RX)
HC-06 RXD ──── ESP32 GPIO21 (TX)
          (3.3V 레벨, 변환 불필요)
```

## 설정 방법 (AT 명령어)

### AT 명령 사용 조건

1. HC-06에 전원을 공급
2. 블루투스로 연결된 기기가 없어야 함 (미연결 상태)
3. Arduino의 UART를 통해 AT 명령 전송
4. **줄바꿈 없이** 명령어 전송 (HC-06은 개행 문자 인식 불가)
5. 명령어 처리 후 약 1~2초 대기

### HC-06 AT 명령어 목록

| 명령어 | 설명 | 응답 |
|--------|------|------|
| `AT` | 통신 확인 | `OK` |
| `AT+VERSION` | 버전 조회 | `OKlinvorV1.8` |
| `AT+NAME` | 이름 조회 (일부 펌웨어) | `OK+NAME:HC-06` |
| `AT+NAMEMyDevice` | 이름 변경 (최대 20자) | `OKsetname` |
| `AT+PIN` | PIN 조회 (일부 펌웨어) | `OK+PIN:1234` |
| `AT+PIN1234` | PIN 변경 (4자리 숫자) | `OKsetPIN` |
| `AT+BAUD1` | 속도 1200bps | `OK1200` |
| `AT+BAUD2` | 속도 2400bps | `OK2400` |
| `AT+BAUD3` | 속도 4800bps | `OK4800` |
| `AT+BAUD4` | 속도 9600bps (기본값) | `OK9600` |
| `AT+BAUD5` | 속도 19200bps | `OK19200` |
| `AT+BAUD6` | 속도 38400bps | `OK38400` |
| `AT+BAUD7` | 속도 57600bps | `OK57600` |
| `AT+BAUD8` | 속도 115200bps | `OK115200` |

> **주의**: HC-06 펌웨어 버전에 따라 응답 형식이 다를 수 있습니다.
> 특히 이름 변경 명령어가 `AT+NAMEMyName` (띄어쓰기 없음)임을 주의.

### AT 명령 Arduino 코드

```cpp
#include <SoftwareSerial.h>

SoftwareSerial bt(10, 11);  // RX=10, TX=11

void setup() {
    Serial.begin(9600);
    bt.begin(9600);  // HC-06 기본 속도
    Serial.println("HC-06 AT 설정 도구");
    Serial.println("명령어를 입력하고 Enter 없이 전송하세요.");
}

void loop() {
    if (Serial.available()) {
        bt.write(Serial.read());
    }
    if (bt.available()) {
        Serial.write(bt.read());
    }
}
```

> **시리얼 모니터 설정**: 줄 마침 없음(No line ending) 선택. 9600bps.

### 이름 변경 예시

```
시리얼 모니터에 입력: AT+NAMEMyArduino
→ HC-06 응답: OKsetname
→ 이후 스마트폰 블루투스 검색 시 "MyArduino"로 표시됨
```

### 전송 속도 변경 예시

```
시리얼 모니터에 입력: AT+BAUD8
→ HC-06 응답: OK115200
→ 이후 HC-06의 UART 속도가 115200bps로 변경됨
→ Arduino 코드의 bt.begin() 값도 115200으로 변경 필수!
```

### 기본 데이터 통신 코드

```cpp
#include <SoftwareSerial.h>

const int BT_RX = 10;
const int BT_TX = 11;
const long BT_BAUD = 9600;

SoftwareSerial bt(BT_RX, BT_TX);

void setup() {
    Serial.begin(9600);
    bt.begin(BT_BAUD);
    pinMode(LED_BUILTIN, OUTPUT);
    Serial.println("HC-06 준비 완료. 스마트폰으로 연결하세요.");
}

void loop() {
    if (bt.available()) {
        char c = bt.read();
        Serial.print("BT 수신: ");
        Serial.println(c);

        // 명령 처리
        switch (c) {
            case '1':
                digitalWrite(LED_BUILTIN, HIGH);
                bt.println("LED ON");
                break;
            case '0':
                digitalWrite(LED_BUILTIN, LOW);
                bt.println("LED OFF");
                break;
        }
    }
}
```

## 주의사항

1. **슬레이브 전용**: HC-06은 항상 슬레이브. Arduino끼리 통신하려면 한쪽은 HC-05(마스터 설정) 필요.
2. **5V 레벨 변환**: HC-05와 동일하게 5V Arduino의 TX → HC-06 RXD에 분압 저항 사용.
3. **AT 명령에 개행 없음**: HC-06은 `\r\n` 줄 바꿈 문자를 AT 명령 구분자로 인식하지 않습니다. 시리얼 모니터에서 "줄 마침 없음" 반드시 설정.
4. **속도 변경 후 코드 수정**: AT+BAUD8로 115200으로 변경했으면 Arduino 코드의 bt.begin()도 115200으로 변경해야 합니다.
5. **연결 중 AT 명령 불가**: 스마트폰이 연결된 상태에서 AT 명령을 보내면 데이터로 처리되어 오동작합니다.
6. **iOS 미지원**: HC-06도 SPP 프로파일이므로 iOS에서 사용 불가.

## 실사용 팁 및 회로 패턴

### HC-05 vs HC-06 선택 기준

```
HC-06 선택:
  - 스마트폰/PC → Arduino 단방향 제어
  - 초보자, 간단한 설정 원할 때
  - 비용 절감이 중요할 때

HC-05 선택:
  - Arduino ↔ Arduino 통신 필요
  - 마스터/슬레이브 동적 전환 필요
  - 더 세밀한 AT 설정 필요
```

### 시리얼 모니터 대체 (블루투스 무선 디버깅)

```cpp
// USB 없이 블루투스로 시리얼 모니터 기능
void loop() {
    // 센서 데이터 전송
    float temp = readTemperature();
    bt.print("온도: ");
    bt.print(temp);
    bt.println("°C");
    delay(1000);
}
// 스마트폰 앱에서 수신하여 실시간 모니터링
```

### 블루투스 연결 상태 감지 (STATE 핀 있는 모델)

```cpp
const int BT_STATE = 8;

void setup() {
    pinMode(BT_STATE, INPUT);
}

bool isBluetoothConnected() {
    return digitalRead(BT_STATE) == HIGH;
}

void loop() {
    if (isBluetoothConnected()) {
        // 연결됨: 데이터 전송
        bt.println("센서 데이터");
    } else {
        // 미연결: 저전력 대기
        delay(1000);
    }
}
```
